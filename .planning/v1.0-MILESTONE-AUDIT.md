## Unsatisfied Requirements (Blockers)

### PROJ-01: sandboxStrategy not persisted
**Phase:** 1 | **Severity:** Blocker

The 3-step project creation wizard captures sandbox strategy in step 2 via a Textarea. Both `createProject` and `updateProject` Zod schemas include `sandboxStrategy: z.string().optional()`. However:
- `Project` model in `prisma/schema.prisma` has no `sandboxStrategy` field
- `createProject` action's `prisma.project.create` data block does not include it
- `updateProject` action's `updateData` object does not include it

User input is silently discarded.

**Fix:** Add `sandboxStrategy String?` to Project model, include in create/update data, run migration.

### WORK-06: AI Story Generation E2E Broken
**Phase:** 3 | **Severity:** Blocker

Individual artifacts for story generation exist (task definition, draft tool, draft cards, context loader) but the cross-phase wiring is broken at 3 points:

1. **epicId discarded:** `epic-detail-client.tsx` calls `initiateStorySession` but only uses `data.conversationId` for redirect — `epicId`/`featureId` never appended as search params
2. **StoryDraftCards orphaned:** Component built at `src/components/work/story-draft-cards.tsx` but has zero import references anywhere in `src/`
3. **STORY_SESSION collapsed:** Chat page collapses `STORY_SESSION` into `TASK_SESSION` prop value — no branch to handle story-specific rendering or forward epicId to API
4. **Prompt never activates:** `/api/chat` route reads `epicId` from body but it's always `undefined` — `buildStorySessionPrompt` never called

**Fix:** Wire epicId through redirect URL → chat page → ChatInterface → API transport body. Import and render StoryDraftCards. Add STORY_SESSION variant to ChatInterface.

## Partial Requirements

### QUES-08: Category Filter Missing
Question filters work for status, scope, priority, and owner. Category filter absent because the `Question` model has no `category` field in the Prisma schema.

### DASH-05: Discovery Dashboard Auto-Refresh Broken
`dashboardSynthesisFunction` triggers on `PROJECT_STATE_CHANGED` event, but no action or route ever sends this event. Dashboard content only updates via manual trigger from PM dashboard.

### ADMIN-01: Jira Sync Web UI Path Broken
Field name mismatch: `stories.ts` fires `STORY_STATUS_CHANGED` with `toStatus` but `jira-sync.ts` destructures `newStatus` — gets `undefined`. The Claude Code API path works correctly (sends `newStatus`). Additionally, `JIRA_SYNC_REQUESTED` event (manual retry) has no Inngest consumer.
