# Phase 5 — UI Design Contract

> Visual and interaction contract for Phase 5: Document Generation, QA, and Administration. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn/ui |
| Preset | base-nova + Neutral base |
| Component library | Radix UI (via shadcn) |
| Icon library | lucide-react 1.7.0 |

### shadcn Components Required (Phase 5)

**Carried from Phases 1-4:** Button, Input, Label, Textarea, Select, Card, Dialog, Sheet, DropdownMenu, Avatar, Badge, Separator, Skeleton, Toast (Sonner), Form, Table, Tabs, Command, ScrollArea, Tooltip, Collapsible, Progress, Popover

**New for Phase 5:** Chart (shadcn Charts wrapping Recharts), Switch (Jira sync toggle), AlertDialog (archive/reactivate confirmations)

| Component | Used By |

| Badge | Jira sync status (Synced/Pending/Failed), defect severity, test result (Pass/Fail/Blocked) |

## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, badge internal padding |
| sm | 8px | Compact element spacing, filter chip padding, table cell padding |
| md | 16px | Default element spacing, card padding, sidebar nav item padding |
| lg | 24px | Section padding, dashboard card internal padding |
| xl | 32px | Layout gaps between major sections |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level vertical spacing |

### Exceptions

| Exception | Value | Justification |
|-----------|-------|---------------|
| Template gallery card min-width | 280px | Cards need room for template preview thumbnail + title + description. Responsive grid with `min(280px, 100%)`. |
| Document preview panel width | 720px | Standard document preview width matching typical A4/Letter proportions. Centered in viewport. |
| Chart container min-height | 200px | Prevents chart collapse on initial render. Standard dashboard chart height. |
| Sidebar width | 240px | Carried from Phase 1. Fixed width. |
| Touch targets | 36px min | Carried from Phase 1. Buttons and interactive elements. |

## Typography

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Metadata | 13px | 400 (regular) | 1.4 | Timestamps, file sizes, sync timestamps, token counts, version dates, chart axis labels |
| Label | 13px | 600 (semibold) | 1.2 | Form labels, column headers, badge text, chart legends, severity labels, filter labels |
| Body | 14px | 400 (regular) | 1.5 | Table cells, defect descriptions, test notes, document descriptions, form inputs |
| Heading | 18px | 600 (semibold) | 1.2 | Card titles, section headings, template names, defect titles |
| Page Title | 24px | 600 (semibold) | 1.2 | Top-level page headings: Documents, Defects, PM Dashboard, Project Settings |

Font stack: `"Inter", -apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif`

Notes:
- Consistent with Phase 2 typography. 4 font sizes: 13px, 14px, 18px, 24px.
- Chart axis labels and tick marks use 13px/400 metadata style in `--muted-foreground`.
- Chart tooltips use 13px/600 label for the metric name, 14px/400 body for the value.
- AI cost dollar amounts in dashboard stat cards use 24px/600 page title for the total, 13px/400 metadata for "this month" or "all time" descriptor.

## Interaction Patterns

### Document Generation Flow

1. PM clicks "Generate" on a template card.
2. Configuration dialog opens: select sections, scope, format.
3. PM clicks "Generate Document" -- dialog transitions to progress view.
4. Inngest step function runs in background. Progress updates via polling.
5. On completion: success toast with download link. Dialog auto-closes.
6. Document appears in version history table.

### Test Execution Recording

1. QA navigates to story detail > QA tab.
2. Clicks "Record Result" -- inline row appears with test case input + result selector.
3. QA types test case description, selects Pass/Fail/Blocked, adds notes.
4. Clicks save (check icon). Row persists. Optimistic update.
5. If Fail selected, "Create Defect" link appears on the row.

### Defect Lifecycle

1. Defect created from failed test or standalone "Create Defect".
2. Status transitions via kanban drag or status dropdown on detail view.
3. Lifecycle: Open > In Progress > Fixed > Verified > Closed.
4. Only QA role can transition to Verified (confirms fix works).
5. Defect detail shows linked story with link to navigate to it.

### Jira Sync Flow

1. PM configures Jira in project settings (URL, token, project key).
2. "Test Connection" validates credentials via API call.
3. On story status change, Inngest fires sync event.
4. Sync status badge updates on the story row (Synced/Pending/Failed).
5. Failed syncs show "Retry" action. Bulk retry available in settings.

### Archive/Reactivate Flow

1. PM navigates to project settings > Project Status section.
2. Clicks "Archive Project" > AlertDialog confirmation > project becomes read-only.
3. All navigation and content remains visible but edit actions are disabled.
4. Visual indicator: 50% opacity on project elements, Lock badge.
5. To reactivate: PM navigates to archived project settings > clicks "Reactivate" > AlertDialog confirmation > project returns to active.
