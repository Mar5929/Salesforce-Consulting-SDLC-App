1. Create `src/lib/inngest/functions/pm-dashboard-synthesis.ts`:
   - Inngest function triggered by `EVENTS.PM_DASHBOARD_SYNTHESIS_REQUESTED`
   - ID: "pm-dashboard-synthesis", retries: 1
   - Aggregates the following data and stores as JSON in a new field on Project (or extends `cachedBriefingContent`):

   **Data structure for cached PM dashboard (per D-14 includes storiesByAssignee, questionsByStatus, knowledgeCoverage):**
   ```typescript
   interface PmDashboardData {
     generatedAt: string
     health: { score: number; label: string }
     storiesByStatus: Record<string, number> // { DRAFT: 5, READY: 3, ... }
     storiesByAssignee: Array<{ assigneeId: string; assigneeName: string; count: number }> // per D-14
     storiesTotal: number
     storiesCompleted: number
     questionsByStatus: Record<string, number> // { OPEN: 10, SCOPED: 5, ANSWERED: 20, ... } per D-14
     knowledgeCoverage: { totalArticles: number; staleCount: number; coveragePercent: number } // per D-14
     sprintVelocity: Array<{ sprintName: string; pointsCompleted: number }>
     aiUsage: {
       totalCost: number
       costOverTime: Array<{ date: string; cost: number }>
       costByArea: Record<string, number> // { transcripts: 5.20, questions: 2.10, ... }
     }
     qa: {
       passCount: number
       failCount: number
       blockedCount: number
       defectsBySeverity: Record<string, number> // { CRITICAL: 2, HIGH: 5, ... }
       openDefectCount: number
     }
     teamActivity: Array<{ memberId: string; memberName: string; action: string; timestamp: string }>
   }
   ```

   - Step 1: Aggregate stories by status via `prisma.story.groupBy({ by: ["status"], _count: true, where: { projectId } })`
   - Step 2: Aggregate stories by assignee via `prisma.story.groupBy({ by: ["assigneeId"], _count: true, where: { projectId } })`, then look up assignee names from ProjectMember
   - Step 3: Aggregate questions by status via `prisma.question.groupBy({ by: ["status"], _count: true, where: { projectId } })`
   - Step 4: Compute knowledge coverage: count total KnowledgeArticle records for project, count stale ones (staleness score > threshold or lastRefreshedAt older than 7 days), compute coveragePercent = ((total - stale) / total) * 100
   - Step 5: Aggregate sprint velocity via `prisma.sprint.findMany` with computed points from completed stories
   - Step 6: Aggregate AI costs from `prisma.sessionLog.findMany({ where: { projectId }, select: { totalInputTokens: true, totalOutputTokens: true, totalCost: true, createdAt: true, sessionType: true } })`
     - Group by date for cost over time (daily)
     - Group by sessionType for cost by area (map sessionType to friendly names: TRANSCRIPT -> "Transcripts", QUESTION_IMPACT -> "Questions", KNOWLEDGE -> "Knowledge", STORY_GENERATION -> "Story Gen", DOCUMENT -> "Doc Gen")
   - Step 7: Aggregate QA data via `prisma.testExecution.groupBy({ by: ["result"], _count: true })` and `prisma.defect.groupBy({ by: ["severity"], _count: true })`
   - Step 8: Fetch recent team activity (latest 20 actions across stories, defects, documents)
   - Step 9: Compute health score (reuse logic from existing dashboard-synthesis or call it)
   - Step 10: Store computed data in `Project.cachedBriefingContent` (extend the existing JSON, add a `pmDashboard` key to avoid overwriting discovery dashboard data):
     ```typescript
     await prisma.project.update({
       where: { id: projectId },
       data: {
         cachedBriefingContent: {
           ...existingContent,
           pmDashboard: dashboardData,
         },
         cachedBriefingGeneratedAt: new Date(),
       },
     })
     ```

   - **Register `synthesizePmDashboard` in `src/app/api/inngest/route.ts`:**
     - Import `synthesizePmDashboard` from the new function file
     - Add to the functions array in the `serve()` call
