```typescript
{storySession && msg.toolInvocations && msg.toolInvocations.length > 0 && (() => {

  if (drafts.length === 0) return null

      drafts={drafts}
      projectId={storySession.projectId}
      epicId={storySession.epicId}
      featureId={storySession.featureId}
    />
  )
})()}
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| URL search params -> server component | epicId/featureId from URL are untrusted user input |
| Client -> /api/chat | epicId/featureId in request body are untrusted |
| AI tool call args -> UI rendering | Tool call arguments rendered in StoryDraftCards |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-01 | Tampering | URL search params (epicId) | accept | epicId is only used to pass to /api/chat body which already validates project membership and conversation ownership. A tampered epicId would load wrong epic's context but user must already be a project member. Low risk -- same trust level as existing request body params. |
| T-07-02 | Spoofing | /api/chat epicId | mitigate | Already mitigated by existing route: auth() checks Clerk session, getCurrentMember() verifies project membership, conversation.findFirst scopes to projectId. No new attack surface. |
| T-07-03 | Information Disclosure | StoryDraftCards rendering AI args | accept | Tool call args are generated by Claude from project context the user already has access to. XSS risk mitigated by React's default escaping of string values in JSX. No dangerouslySetInnerHTML used. |
</threat_model>
