1. Accepts `{ projectId, epicId?, featureId? }` params
2. Loads from the project:

   - Answered questions scoped to the epic/feature -- include questionText and answerText
   - Decisions scoped to the epic/feature (via DecisionScope join) -- include title and rationale

   ...

**Draft tool (src/lib/agent-harness/tools/create-story-draft.ts) -- NEW FILE:**
Follow the pattern from `create-question.ts`. Define a tool that the AI calls to propose a story draft:
- Tool name: `create_story_draft`
- Description: "Propose a new user story draft for the user to review. Include title, persona, description, acceptance criteria, estimated story points, priority, and impacted Salesforce components."
- Input schema (Zod):
  ```typescript
  z.object({
    title: z.string().describe("Story title"),
    persona: z.string().optional().describe("User persona, e.g. 'Sales Manager'"),
    description: z.string().describe("Story description in user story format"),
    acceptanceCriteria: z.string().describe("Acceptance criteria in Given/When/Then format"),
    storyPoints: z.number().int().min(1).max(13).optional().describe("Fibonacci estimate: 1,2,3,5,8,13"),
    priority: z.enum(["LOW", "MEDIUM", "HIGH", "CRITICAL"]).default("MEDIUM"),
    components: z.array(z.object({
      componentName: z.string().describe("Salesforce component API name"),
      impactType: z.enum(["CREATE", "MODIFY", "DELETE"]),
    })).optional().describe("Impacted Salesforce components"),
    reasoning: z.string().describe("Why this story is needed based on the requirements/context"),
  })
  ```
- Handler: Stores the draft in the ChatMessage.toolCalls JSON field (same pattern as transcript extraction). Returns `{ success: true, draftId: generatedCuid }`. The draft is NOT persisted to the Story table until the user accepts it.

- Components list: each as a pill showing `componentName (IMPACT_TYPE)`

**Initiate story session (src/actions/conversations.ts) -- MODIFY EXISTING FILE:**
Add an `initiateStorySession` server action that creates a Conversation record with type `STORY_SESSION`:
```typescript
export const initiateStorySession = actionClient.schema(z.object({
  projectId: z.string(),
  epicId: z.string(),
  featureId: z.string().optional(),
})).action(async ({ parsedInput, ctx }) => {
  const db = scopedPrisma(parsedInput.projectId)
  const conversation = await db.conversation.create({
    data: {
      projectId: parsedInput.projectId,
      type: "STORY_SESSION",
      title: "AI Story Generation",
      createdById: ctx.userId,
      metadata: {
        epicId: parsedInput.epicId,
        featureId: parsedInput.featureId || null,
      },
    },
  })
  return { conversationId: conversation.id }
})
```
This action is called by the "Generate Stories" menu item on the epic detail page (Plan 02 Task 2). The returned `conversationId` is used to redirect to the chat page: `/projects/{projectId}/chat/{conversationId}`.
