**Sprint server actions (src/actions/sprints.ts) -- NEW FILE:**
Follow the `actionClient` + `scopedPrisma` pattern from existing actions.

1. `createSprint` -- schema: `{ projectId: z.string(), name: z.string().min(1).max(100), goal: z.string().optional(), startDate: z.string().datetime(), endDate: z.string().datetime() }`. Per D-10, no capacity field. Validate endDate > startDate. Create with status PLANNING.

2. `updateSprint` -- schema: `{ projectId: z.string(), sprintId: z.string(), name: z.string().optional(), goal: z.string().optional(), startDate: z.string().datetime().optional(), endDate: z.string().datetime().optional(), status: z.nativeEnum(SprintStatus).optional() }`. Validates sprint belongs to project.

3. `getSprints` -- schema: `{ projectId: z.string() }`. Returns all sprints with `_count: { stories: true }`, plus computed `totalPoints` via a raw aggregate or by loading stories and summing. Include `stories: { select: { storyPoints: true, status: true } }` so the client can compute totals. Order by `startDate desc`.

4. `getSprint` -- schema: `{ projectId: z.string(), sprintId: z.string() }`. Returns sprint with stories including their storyComponents, assignee, feature relations.

5. `assignStoriesToSprint` -- schema: `{ projectId: z.string(), sprintId: z.string(), storyIds: z.array(z.string()).min(1) }`. Per D-09 and Pitfall 3:
   ```typescript
   await db.$transaction(async (tx) => {
     for (const storyId of parsedInput.storyIds) {
       const story = await tx.story.findUniqueOrThrow({ where: { id: storyId } })
       // Auto-transition READY -> SPRINT_PLANNED (D-09)
       // If not READY, keep current status (Pitfall 3: don't block assignment)
       const newStatus = story.status === "READY" ? "SPRINT_PLANNED" : story.status
       await tx.story.update({
         where: { id: storyId },
         data: { sprintId: parsedInput.sprintId, status: newStatus },
       })
     }
   })
   // Fire event for conflict detection (D-19)
   await inngest.send({
     name: EVENTS.SPRINT_STORIES_CHANGED,
     data: { sprintId: parsedInput.sprintId, projectId: parsedInput.projectId },
   })
   ```

6. `removeStoriesFromSprint` -- schema: `{ projectId: z.string(), storyIds: z.array(z.string()).min(1) }`. Sets `sprintId: null` and if status is SPRINT_PLANNED, reverts to READY. Fires SPRINT_STORIES_CHANGED.
