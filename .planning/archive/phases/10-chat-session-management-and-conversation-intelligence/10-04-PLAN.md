Create `src/lib/agent-harness/context/smart-retrieval.ts`:

```typescript
/**
 * Smart two-pass context retrieval for general chat.
 * Pass 1: Load compact summaries from all project data sources.
 * Pass 2: Semantic search to find most relevant full content for the user's question.
 * Stays within token budget.
 */
```

**Constants:**
```typescript
const TOKEN_BUDGET = 6000
const BUDGET_PROJECT_SUMMARY = 500   // tokens
const BUDGET_SUMMARIES = 1500        // tokens for all summary layers
const BUDGET_FULL_CONTENT = 4000     // tokens for top-K relevant full items
```

**Summary loader functions (Pass 1):**
Each returns compact text within its sub-budget.

1. `getStorySummaries(projectId: string)`: Query stories via scopedPrisma, select id, title, status, priority, storyPoints. Format as compact lines: "[STATUS] Title (priority, Xpts)". Take top 30 sorted by updatedAt DESC.

2. `getSprintSummary(projectId: string)`: Query active sprint(s) via scopedPrisma where status is PLANNING or ACTIVE. Format: "Sprint: {name}, {startDate}-{endDate}, {storyCount} stories, Status: {status}".

3. `getOrgKnowledgeSummary(projectId: string)`: Query OrgComponent count by type, DomainGrouping count, BusinessProcess count. Format as: "Org Knowledge: X components (Y Apex, Z Flow, ...), N domain groupings, M business processes".

4. `getTranscriptSummary(projectId: string)`: Query transcripts via scopedPrisma, select id, title, createdAt. Take top 10. Format: "Transcripts: {title} ({date}), ...".

5. `getDecisionSummary(projectId: string)`: Query decisions via scopedPrisma, select id, title, rationale. Take top 15 by createdAt DESC. Format as lines: "- {title}: {rationale truncated to 100 chars}".

6. `getQuestionSummary(projectId: string)`: Query questions via scopedPrisma grouped by status count. Format: "Questions: X open, Y scoped, Z answered, ...".

**assembleSmartChatContext function:**
```typescript
export async function assembleSmartChatContext(

  userMessage: string
```

Interface:
```typescript
export interface SmartChatContext {
  projectSummary: string
  summaries: {
    stories: string
    sprint: string
    orgKnowledge: string
    articles: string
    transcripts: string
    decisions: string
    questions: string
  }
  relevantFullContent: string
  totalTokenEstimate: number
}
```

Implementation:
1. **Pass 1** — load all summaries in parallel via Promise.all:
   - getProjectSummary(projectId)
   - getStorySummaries(projectId)
   - getSprintSummary(projectId)
   - getOrgKnowledgeSummary(projectId)
   - getArticleSummaries(projectId) — format as "{title}: {summary}" lines
   - getTranscriptSummary(projectId)
   - getDecisionSummary(projectId)
   - getQuestionSummary(projectId)

2. **Pass 2** — semantic search for user message:
   - Call `globalSearch(projectId, userMessage, { limit: 5 })` to get top 5 semantically relevant items
   - Flatten grouped results into a ranked list
   - For each result, format as: "## {entityType}: {title}\n{snippet}" 
   - If globalSearch returns no results (embeddings not available), fall back to including more summary data
   - Trim full content to fit within BUDGET_FULL_CONTENT using charsToTokens

3. **Assemble** — combine summaries and full content, truncating each section to fit its budget using charsToTokens for estimation.

4. **Return** SmartChatContext with all sections + totalTokenEstimate
