## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, badge internal spacing |
| sm | 8px | Conversation row internal padding, tab bar gaps, card field spacing |
| md | 16px | Conversation sidebar padding, message area padding, section gaps |
| lg | 24px | Sidebar section breaks, enrichment card group spacing |
| xl | 32px | Layout gap between conversation sidebar and chat area |
| 2xl | 48px | Empty state vertical centering offset |
| 3xl | 64px | Page-level spacing (unused in this phase) |

Exceptions: Send button and textarea input use 44px min-height for touch target compliance (existing pattern from `chat-interface.tsx`). Conversation sidebar width is 320px (fixed).

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `hsl(0 0% 100%)` / `--background` | Chat message area, page surface, conversation detail pane |
| Secondary (30%) | `hsl(0 0% 98%)` / `--muted` | Conversation sidebar background, input area, enrichment suggestion card backgrounds, tab bar background strip |

## Conversation Sidebar Design Contract

### Layout (D-01)

```
+------------------------------------------+-------------------------------------------+
| Conversation Sidebar (320px)             | Chat Area (flex-1)                        |
| +--------------------------------------+ |                                           |
| | [+ New]              [Search icon]   | | [Header: title + token display]           |
| +--------------------------------------+ |                                           |
| | All | Stories | Briefings | Trans | Q | | [Message list / empty state]              |
| +--------------------------------------+ |                                           |
| | --- Pinned ---                       | |                                           |
| | [*] General Chat          2m ago     | |                                           |
| | --- Sessions ---                     | |                                           |
| | [S] Story Gen: User Mgmt  1h ago    | |                                           |
| | [B] Daily Briefing         3h ago   | |                                           |
| | [T] Kickoff Transcript     1d ago   | | [Input area]                              |
| | [Q] Auth Flow Questions    2d ago   | |                                           |
| +--------------------------------------+ |                                           |
+------------------------------------------+-------------------------------------------+
```

- Sidebar background: `bg-muted` (`hsl(0 0% 98%)`)
- Sidebar border-right: `border-border` (`hsl(0 0% 90%)`)
- Sidebar width: 320px fixed. No collapse behavior in V1 (desktop-only app per requirements).
- General chat is always pinned at top, separated by `Separator` from session list

### Conversation Row (D-04)

- Height: auto, min 48px
- Left: Session type icon (16px, `text-muted-foreground`) using lucide icons:
  - GENERAL_CHAT: `MessageSquare`
  - STORY_SESSION: `BookOpen`
  - TRANSCRIPT_SESSION: `FileText`
  - BRIEFING_SESSION: `LayoutDashboard`
  - QUESTION_SESSION: `HelpCircle`
  - ENRICHMENT_SESSION: `Sparkles`
- Center: Title (14px regular, single line truncated with `truncate`), subtitle line with relative timestamp (13px `text-muted-foreground`)
- Right: `SessionStatusBadge` for non-ACTIVE sessions only. ACTIVE sessions show no badge.
- Selected row: `bg-background` (white) with left 2px accent border (`border-l-2 border-primary`)
- Hover: `bg-background/80`, reveals action icons (Pencil for rename, Archive for archive) at 14px `text-muted-foreground`
- Right-click: DropdownMenu with Rename, Archive options (no delete per D-12 — archive only)

### Tab Bar (D-02)

- Uses shadcn `Tabs` component with `variant="default"`
- Tabs: All, Stories, Briefings, Transcripts, Questions
- Tab text: 13px regular. Active tab: 13px semibold with accent underline
- Tab overflow: horizontally scrollable on narrow viewports (unlikely at 320px but safe)

### Search (D-03)

- `Input` component with `Search` lucide icon (16px) as left adornment
- Placeholder: "Search conversations..."
- 13px input text
- Positioned below "+ New" button, above tab bar
- Filters conversation list immediately on input (client-side filter of loaded data)

### "+ New" Button (D-05)

- `Button` variant="default" size="sm" with `Plus` icon (16px) + "New Session" text
- Positioned top-right of sidebar header
- Opens `DropdownMenu` with session type options:
  - "Story Generation" — then prompts for epic selection
  - "Briefing" — creates immediately, navigates to session
  - "Question Impact" — then prompts for question selection
  - "Story Enrichment" — then prompts for story selection
- Each dropdown item: session type icon (16px) + label (14px) + description (13px `text-muted-foreground`)

## Session Type Visual Contracts

### BRIEFING_SESSION (D-07)

- Entry point: "Generate Briefing" button on discovery dashboard
- Button: `Button` variant="default" with `LayoutDashboard` icon (16px) + "Generate Briefing"
- Session opens in chat area. AI streams briefing content.
- Briefing output rendered as structured markdown in `BriefingOutput` component
- BriefingOutput layout: Card with sections for "Current Focus", "Outstanding Items", "Recommended Actions"
- Each section: 16px semibold heading + 14px body content
- Session auto-completes after briefing generation (D-10). Input area shows read-only notice: "Briefing complete. Start a new session to ask follow-up questions."

### QUESTION_SESSION (D-08)

- Entry point: Automatic trigger from Inngest when contradiction/complex impact detected
- If background-only (simple answer): no chat UI needed, notification sent with result
- If interactive (contradiction): session created, user notified via in-app notification with link to chat session
- Chat area shows standard message flow. AI presents the contradiction and asks for resolution.
- No unique card components. Standard message bubbles with AI text explaining the conflict.

### ENRICHMENT_SESSION (D-09)

- Entry point: "Enrich with AI" button on story detail page
- Button: `Button` variant="outline" with `Sparkles` icon (16px) + "Enrich with AI"
- Session opens in chat area. AI analyzes story context and streams suggestions.
- Suggestions rendered as `EnrichmentSuggestionCards` — same card-based accept/reject as StoryDraftCards pattern:

```
+-------------------------------------------------------+
| [Sparkles icon] Suggestion: [Category]     [Accept][X] |
+-------------------------------------------------------+
| Field: [field name]                                     |
| Current: [current value or "empty"]                     |
| Suggested: [AI suggestion text]                         |
+-------------------------------------------------------+
```

- Card background: `bg-muted/50` with `border border-border rounded-lg`
- Category examples: "Acceptance Criteria", "Impacted Components", "Technical Notes"
- Accept button: `Button` variant="default" size="sm" with `Check` icon
- Reject button: `Button` variant="ghost" size="sm" with `X` icon
- Accepted cards: collapse to single line "Applied: [category]" with `Check` icon in `text-chart-2` (green)
- Rejected cards: collapse to single line "Dismissed: [category]" in `text-muted-foreground`
- Session auto-completes when all suggestions are accepted/rejected (D-10)

## Session Lifecycle Visual States (D-10, D-11, D-12)

### SessionStatusBadge

| Status | Badge Variant | Color | Text |
|--------|--------------|-------|------|
| ACTIVE | `outline` | default border | "Active" |
| COMPLETE | `secondary` | `bg-secondary text-secondary-foreground` | "Complete" |
| FAILED | custom | `bg-destructive/10 text-destructive border-destructive/20` | "Failed" |

- ACTIVE badge is only shown in conversation row for task sessions, not general chat
- COMPLETE badge is always shown
- FAILED badge includes a "Retry" link (13px `text-primary hover:underline`) adjacent to the badge (D-11)

### Failed Session Retry (D-11)

1. User clicks "Retry" on a FAILED session row
2. New session of the same type created with same context parameters
3. Old session becomes read-only (input area disabled, notice: "This session failed. A retry session has been created.")
4. User is navigated to the new session
5. Old session remains in conversation list for audit trail, with "Failed" badge

### Archive Flow (D-12)

- Archive action available via conversation row hover/right-click menu
- Clicking "Archive" opens AlertDialog:
  - Title: "Archive conversation?"
  - Body: "This conversation will be hidden from your list. You can restore it anytime from the archived filter."
  - Cancel: `Button` variant="outline"
  - Confirm: `Button` variant="default" text "Archive Conversation"
- Archived conversations hidden from default list view
- "Show Archived" toggle at bottom of conversation sidebar: 13px `text-muted-foreground` link
- When toggled on: archived rows show with 50% opacity and "Unarchive" action replacing "Archive"

## Interaction Contracts

### Conversation Selection Flow

1. User lands on `/projects/[id]/chat` — sidebar loads, general chat selected by default
2. User clicks a conversation row — URL updates to `/projects/[id]/chat/[conversationId]`, chat area loads that session
3. Back to general chat: click pinned general chat row at top

### Session Creation via "+ New" (D-05)

1. User clicks "+ New" button in sidebar
2. DropdownMenu opens with session type options
3. User selects a type (e.g., "Story Enrichment")
4. If context needed: inline select/search for the target entity (story, epic, question)
5. Session created via server action, conversation row added to sidebar, chat area navigates to new session

### Session Creation via Source Page (D-05, D-07, D-09)

1. User clicks "Generate Briefing" on dashboard OR "Enrich with AI" on story detail
2. Server action creates the session with pre-filled context
3. User is redirected to `/projects/[id]/chat/[newConversationId]`
4. Sidebar updates to show the new session selected

### Session Auto-Complete (D-10)

1. Task session reaches completion condition (briefing generated, all enrichment suggestions resolved, question impact assessed)
2. Session status updates to COMPLETE via server action
3. SessionStatusBadge updates in sidebar to "Complete"
4. Input area transitions to read-only with completion notice
5. User can still scroll and read the session history

### Rename Conversation

1. User hovers conversation row, clicks pencil icon (or right-click > Rename)
2. Title text becomes an inline editable input (13px, same width as title area)
3. Enter to save, Escape to cancel
4. Server action updates conversation title

## Accessibility

- Conversation sidebar uses `role="navigation"` with `aria-label="Conversation list"`
- Each conversation row uses `role="option"` within a `role="listbox"` container
- Selected conversation row has `aria-selected="true"`
- Tab bar uses shadcn Tabs which provides keyboard navigation (arrow keys) and ARIA roles natively
- Search input has `aria-label="Search conversations"`
- "+ New" dropdown trigger has `aria-label="Create new session"`
- Session status badges use `aria-label="Session status: [status]"`
- Archive AlertDialog uses native shadcn AlertDialog accessibility (focus trap, Escape to close)
- Enrichment suggestion cards: Accept button has `aria-label="Accept suggestion: [category]"`, Reject has `aria-label="Dismiss suggestion: [category]"`
- Inline rename uses `aria-label="Rename conversation"` on the input
- Keyboard: Tab moves between sidebar and chat area. Within sidebar, arrow keys navigate rows. Enter selects.
