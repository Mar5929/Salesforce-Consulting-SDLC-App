   - Export `orgClassifyTask` as a TaskDefinition
   - System prompt: "You are a Salesforce consulting architect. Analyze these org components and group them into logical domain areas. For each domain grouping, provide: name (e.g., 'Sales Pipeline', 'Case Management', 'Partner Portal'), description, list of component API names that belong to this domain, and your confidence (0-100) that this grouping is correct. Focus on business meaning, not technical structure."
   - Token budget: { input: 15000, output: 4000 }
   - Output schema: Zod array of `{ name: string, description: string, componentApiNames: string[], confidence: number }`
   - No tools needed (pure classification)

2. Create `src/lib/agent-harness/tasks/org-synthesize.ts`:
   - Export `orgSynthesizeTask` as a TaskDefinition
   - System prompt: "You are a Salesforce consulting architect. For the given domain grouping and its components, identify the business processes they support. For each process: provide name, description, complexity (LOW/MEDIUM/HIGH), and map each component with its role (e.g., 'primary data store', 'automation trigger', 'validation enforcement'). Also identify dependencies between processes."
   - Token budget: { input: 10000, output: 3000 }
   - Output schema: Zod array of `{ name: string, description: string, complexity: "LOW"|"MEDIUM"|"HIGH", components: Array<{ apiName: string, role: string, isRequired: boolean }>, dependsOn: string[] }`

3. Create `src/lib/inngest/functions/org-ingestion.ts`:
   - Register as Inngest function: `inngest.createFunction({ id: "org-ingestion", concurrency: [{ scope: "event", key: "event.data.projectId", limit: 1 }] }, { event: EVENTS.ORG_INGESTION_REQUESTED }, ...)`
   - **Step 1 — Parse:** Query all active OrgComponent records for the project. Group by componentType. Build a structured summary: `{ objects: [...], apexClasses: [...], flows: [...], ... }` with apiName and label for each. Store in step result.
   - **Step 2 — Classify:** Batch components into chunks of max 50. For each batch, call `executeTask(orgClassifyTask, { components: batch })`. Collect all domain grouping suggestions. Create DomainGrouping records with isAiSuggested=true, isConfirmed=false. Assign each OrgComponent.domainGroupingId based on the AI's componentApiNames mapping. Track confidence as a field on the DomainGrouping (add `confidence Float?` to schema if not present — check first, if not available store in description or as JSON).
   - **Step 3 — Synthesize+Articulate:** For each DomainGrouping created in Step 2, query its OrgComponent records. Call `executeTask(orgSynthesizeTask, { domainGrouping: { name, description, components } })`. Create BusinessProcess records with isAiSuggested=true, isConfirmed=false. Create BusinessProcessComponent join records with role and isRequired. Set BusinessProcess.domainGroupingId.
   - **Step 4 — Finalize:** Update Project metadata (e.g., a `lastIngestionAt` timestamp if desired). Log completion.

4. Register the function in the Inngest serve handler (add to `src/app/api/inngest/route.ts`).
