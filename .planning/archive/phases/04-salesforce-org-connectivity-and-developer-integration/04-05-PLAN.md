1. Create a shared helper `src/app/api/v1/_lib/api-handler.ts` (or inline in each route):
```typescript
import { validateApiKey } from "@/lib/api-keys/middleware"
import { checkRateLimit } from "@/lib/api-keys/rate-limit"
import { NextResponse } from "next/server"

export async function withApiAuth(
  request: Request,
  handler: (projectId: string, apiKeyId: string) => Promise<NextResponse>
): Promise<NextResponse> {
  const authResult = await validateApiKey(request)
  if (authResult instanceof NextResponse) return authResult

  const { projectId, apiKeyId } = authResult
  const endpoint = new URL(request.url).pathname
  const rateResult = await checkRateLimit(apiKeyId, endpoint)
  if (!rateResult.allowed) {
    return NextResponse.json({ error: "Rate limit exceeded" }, {
      status: 429,
      headers: { "Retry-After": String(rateResult.retryAfter ?? 60) }
    })
  }

  return handler(projectId, apiKeyId)
}
```

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-14 | Spoofing | API key auth | mitigate | bcrypt hash comparison, keyPrefix index for fast rejection, 401 on missing/invalid |
| T-04-15 | Denial of Service | API rate abuse | mitigate | Postgres sliding window rate limiter, 429 with Retry-After header |
| T-04-16 | Elevation of Privilege | Cross-project API access | mitigate | API key scoped to project, all queries filter by projectId from API key |
| T-04-17 | Tampering | Invalid story status transition | mitigate | Status machine validation rejects illegal transitions with 400 |
| T-04-18 | Information Disclosure | API key shown once | mitigate | Raw key never stored, only bcrypt hash. Shown once at creation, warning text displayed. |
