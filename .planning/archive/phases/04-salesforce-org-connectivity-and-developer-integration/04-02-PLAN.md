  <action>
1. Create `src/app/api/auth/salesforce/authorize/route.ts`:
   - Export async GET handler
   - Extract `projectId` from searchParams
   - Verify user is authenticated via Clerk `auth()` — return 401 if not
   - Verify user has SA or PM role on the project (query ProjectMember where clerkUserId matches and role IN ["SA", "PM"]) — return 403 if not (per D-05)
   - Verify project doesn't already have a connected org (sfOrgInstanceUrl not null) — return 400 with message "Org already connected. Disconnect first." (per D-03)
   - Call `buildAuthorizationUrl(projectId)` from oauth.ts
   - Return `redirect(authUrl)` using `next/navigation`

2. Create `src/app/api/auth/salesforce/callback/route.ts`:
   - Export async GET handler
   - Extract `code` and `state` (projectId) from searchParams
   - If missing code or state: redirect to `/projects/${projectId}/settings/org?error=oauth_failed`
   - Call `exchangeCodeForTokens(code, projectId)` from oauth.ts
   - On success: update Project with `sfOrgInstanceUrl`, `sfOrgAccessToken`, `sfOrgRefreshToken`
   - Send Inngest event `EVENTS.ORG_SYNC_REQUESTED` with `{ projectId, syncType: "FULL" }`
   - Redirect to `/projects/${projectId}/settings/org?connected=true`
   - On error: redirect to `/projects/${projectId}/settings/org?error=oauth_failed`
   - Note: use `(dashboard)` route group to match existing app structure: redirect URL = `${NEXT_PUBLIC_APP_URL}/projects/${projectId}/settings/org?...`

3. Create `src/actions/org-connection.ts`:
   - Use `actionClient` from safe-action.ts
   - `disconnectOrg` action: Zod input `{ projectId: z.string() }`. Verify SA/PM role. Clear sfOrgInstanceUrl, sfOrgAccessToken, sfOrgRefreshToken on Project (set to null). Do NOT delete OrgComponent/DomainGrouping/BusinessProcess records (per D-01 disconnect confirmation copy). Return `{ success: true }`.
   - `requestSync` action: Zod input `{ projectId: z.string(), syncType: z.enum(["FULL", "INCREMENTAL"]) }`. Verify SA/PM role. Verify sfOrgInstanceUrl is not null (org connected). Send Inngest event `EVENTS.ORG_SYNC_REQUESTED` with `{ projectId, syncType }`. Return `{ success: true }`.
   - `getOrgConnectionStatus` action: Zod input `{ projectId: z.string() }`. Query Project for sfOrgInstanceUrl, sfOrgLastSyncAt, sfOrgAccessToken. Query OrgComponent count where projectId matches. Return `{ instanceUrl, lastSyncedAt, componentCount, isConnected: !!sfOrgInstanceUrl, needsReauth: !!sfOrgInstanceUrl && !sfOrgAccessToken }`.
  </action>

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-04-05 | Spoofing | OAuth callback | mitigate | Validate state parameter contains valid projectId, verify user session via Clerk auth() |
| T-04-06 | Elevation of Privilege | Connect/disconnect actions | mitigate | SA/PM role check in both API routes and server actions (D-05) |
| T-04-07 | Information Disclosure | Token in callback URL | mitigate | Auth code is single-use and short-lived; tokens never exposed in URLs |
