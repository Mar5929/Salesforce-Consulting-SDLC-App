1. Create `src/lib/salesforce/metadata-sync.ts`:

```typescript
export async function syncMetadataType(
  conn: jsforce.Connection,
  projectId: string,
  typeConfig: MetadataTypeConfig,
  sinceDate?: Date
): Promise<{ added: number; modified: number }>
```
   - Call `conn.metadata.list([{ type: typeConfig.type }])` (or with folder for certain types)
   - If sinceDate: filter results where lastModifiedDate > sinceDate (with 5-minute buffer: `new Date(sinceDate.getTime() - 5 * 60 * 1000)`)
   - Map Salesforce component type to ComponentType enum: "CustomObject" -> OBJECT, "ApexClass" -> APEX_CLASS, "ApexTrigger" -> APEX_TRIGGER, "LightningComponentBundle" -> LWC, "Flow" -> FLOW, "PermissionSet" -> PERMISSION_SET, "Profile" -> PROFILE, "CustomLabel" -> OTHER (or add CUSTOM_LABEL), "CustomSetting" -> OTHER, "CustomMetadata" -> OTHER
   - For each component: `prisma.orgComponent.upsert` with where clause `{ projectId_apiName_componentType: { projectId, apiName: item.fullName, componentType } }`, create with label, componentType, lastSyncedAt=new Date(), update with label, lastSyncedAt, isActive=true
   - If typeConfig.describeFields (CustomObject only): call `conn.sobject(item.fullName).describe()`, upsert each field as child OrgComponent with componentType=FIELD, apiName=`${objectName}.${field.name}`, parentComponentId pointing to the object's OrgComponent id
   - For lookup/masterDetail fields: create OrgRelationship with sourceComponentId=field's component, targetComponentId=referenced object's component (look up by apiName), relationshipType=LOOKUP or MASTER_DETAIL

```typescript
export async function syncMetadata(
  projectId: string,
  syncType: "FULL" | "INCREMENTAL"
): Promise<{ added: number; modified: number; removed: number; byType: Record<string, number> }>
```
   - Fetch project for sfOrgLastSyncAt
   - Create OrgSyncRun record with status=RUNNING
   - Get jsforce connection via getSalesforceConnection(projectId)
   - Determine sinceDate: null for FULL, sfOrgLastSyncAt for INCREMENTAL
   - Iterate SUPPORTED_METADATA_TYPES, call syncMetadataType for each
   - After all types: for FULL sync, mark any OrgComponent where lastSyncedAt < syncRun.startedAt as isActive=false (removed)
   - Update Project.sfOrgLastSyncAt to new Date()
   - Update OrgSyncRun with status=COMPLETED, componentCounts, completedAt
   - On error: update OrgSyncRun with status=FAILED, errorMessage
   - Return counts

2. Create `src/lib/inngest/functions/metadata-sync.ts`:
   - Register as Inngest function: `inngest.createFunction({ id: "metadata-sync", concurrency: [{ scope: "event", key: "event.data.projectId", limit: 1 }] }, { event: EVENTS.ORG_SYNC_REQUESTED }, ...)`
   - Extract projectId, syncType from event.data
   - Use `step.run("sync-metadata", () => syncMetadata(projectId, syncType))`
   - On completion: `step.sendEvent("notify-sync-complete", { name: EVENTS.ORG_SYNC_COMPLETED, data: { projectId, counts } })`
   - On failure: the Inngest retry mechanism handles retries (3 attempts default)
