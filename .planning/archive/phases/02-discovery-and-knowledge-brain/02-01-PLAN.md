<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| AI output -> DB | AI-generated content crosses into persistent storage |
| AI tool inputs -> DB | Tool call arguments from Claude cross into Prisma queries |
| User input -> AI prompt | User messages become part of Claude system/user prompts |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | tool-executor.ts | mitigate | sanitizeToolInput() with DOMPurify on ALL string inputs before DB write |
| T-02-02 | Tampering | engine.ts | mitigate | Use Prisma parameterized queries exclusively ($queryRaw tagged templates). Ban $queryRawUnsafe. |
| T-02-03 | Information Disclosure | engine.ts | mitigate | Never include API keys, env vars, or internal paths in system prompts. Context assembly controls what data enters prompts. |
| T-02-04 | Denial of Service | engine.ts | mitigate | maxIterations cap on agent loops prevents runaway API calls. maxRetries cap prevents infinite retry. |
| T-02-05 | Elevation of Privilege | tool-executor.ts | mitigate | All tool DB operations use scopedPrisma(projectId) — tools cannot access cross-project data |
</threat_model>
