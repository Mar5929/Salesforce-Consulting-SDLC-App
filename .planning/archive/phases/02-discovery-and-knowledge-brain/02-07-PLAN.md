    1. Create `prisma/migrations/tsvector-setup.sql` per tech spec Section 8.1:
       - Add search_vector tsvector columns to Question, Decision, KnowledgeArticle, Requirement, Risk tables
       - Create GIN indexes on each search_vector column
       - Create trigger functions that auto-populate search_vector on INSERT and UPDATE:
         - Question: tsvector from questionText || ' ' || COALESCE(answer, '')
         - Decision: tsvector from title || ' ' || COALESCE(outcome, '') || ' ' || COALESCE(rationale, '')
         - KnowledgeArticle: tsvector from title || ' ' || COALESCE(content, '')
         - Requirement/Risk: tsvector from title || ' ' || COALESCE(description, '')
       - Run via: `npx prisma db execute --file prisma/migrations/tsvector-setup.sql`
       - These are manual SQL migrations outside Prisma schema (Prisma can't express tsvector — see pitfall 3)

    2. **IMPORTANT — Vector dimension migration:** The Prisma schema has `vector(1536)` for embedding columns. Voyage AI's default dimension is 1024. Per user decision to use Voyage AI, create an additional SQL migration to alter embedding columns:
       - `ALTER TABLE "Question" ALTER COLUMN embedding TYPE vector(1024);`
       - `ALTER TABLE "KnowledgeArticle" ALTER COLUMN embedding TYPE vector(1024);`
       - (And any other tables with embedding columns — check schema for all Unsupported("vector(1536)") occurrences)
       - Also update the Prisma schema comments to note `vector(1024)` for Voyage AI
       - Create HNSW indexes on the embedding columns for efficient similarity search

    3. Create `src/lib/search/embeddings.ts`:
       - Use Voyage AI API (REST HTTP calls, no heavy SDK needed):
         ```typescript
         async function generateEmbedding(text: string): Promise<number[]>
         async function generateEmbeddings(texts: string[]): Promise<number[][]>
         ```
       - Model: `voyage-3-large` (Anthropic-recommended)
       - Dimensions: 1024 (Voyage AI default for voyage-3-large)
       - API endpoint: `https://api.voyageai.com/v1/embeddings`
       - Use `process.env.VOYAGE_API_KEY`
       - Add VOYAGE_API_KEY to env.ts validation (optional — embedding generation is non-critical, search gracefully falls back to full-text when embeddings not available)
       - Batch support: Voyage API accepts up to 128 texts per request
       - Return number[] arrays matching vector(1024) dimension

    4. Create `src/lib/search/global-search.ts` per D-21, D-22:
       - `globalSearch(projectId: string, query: string, options?: { types?, limit? })`:
         a. **Layer 1 — Filtered search (SRCH-01):** If query looks like structured filter (e.g., "status:open category:technical"), parse and build Prisma where clause. Return exact matches.
         b. **Layer 2 — Full-text search (SRCH-02):** Run tsvector query via `prisma.$queryRaw` with tagged template literals (NEVER $queryRawUnsafe). Query across Question, Decision, KnowledgeArticle, Requirement, Risk tables. Use ts_headline for result snippets and ts_rank for relevance scoring. Per tech spec Pattern 5.
         c. **Layer 3 — Semantic search (SRCH-03):** Generate embedding for query text via generateEmbedding(). Run pgvector cosine similarity query across all tables with embedding columns. Return top results by similarity score.
         d. Merge results from all three layers, deduplicate by entity ID, rank by combined score (full-text rank * weight + semantic similarity * weight).
         e. Return grouped results: { questions: [], articles: [], decisions: [], requirements: [], risks: [] }
         f. Each result: { id, displayId, title, snippet, entityType, score, status?, stalenessBadge? }
       - Graceful degradation: If embeddings not available (VOYAGE_API_KEY not set or embedding columns empty), skip semantic layer and return full-text + filtered results only.

    5. Create `src/actions/search.ts` using next-safe-action:
       - `search(projectId, query, types?, limit?)`: Wraps globalSearch with auth check and project scope validation
