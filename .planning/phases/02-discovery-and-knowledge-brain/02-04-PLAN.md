    1. Create `src/lib/display-id.ts`:
       - `generateDisplayId(projectId, entityType: 'Question' | 'Decision' | 'Risk' | 'Requirement', prisma)`:
         a. Query MAX displayId for the entity type + project (e.g., "Q-15" -> extract 15)
         b. Increment: next = max + 1
         c. Prefix: Q- for Question, D- for Decision, R- for Risk, REQ- for Requirement
         d. Wrap in transaction. On unique constraint violation, retry once with next number.
         e. Return formatted string (e.g., "Q-16")

    2. Create `src/actions/questions.ts` using next-safe-action pattern (D-01, D-03):
       - `createQuestion`: Zod schema with projectId, questionText (min 1), category (QuestionCategory enum), scope (QuestionScope enum), priority (Priority enum), scopeEpicId?, scopeFeatureId?, assigneeId?. Generate displayId. Create Question record. Emit NOTIFICATION_SEND event if assignee set. Return question.
       - `updateQuestion`: Partial update. Status transitions validated per D-03 lifecycle (Open > Scoped > Owned > Answered > Reviewed). Emit ENTITY_CONTENT_CHANGED event.
       - `answerQuestion`: Takes questionId, answerText, sourceType, sourceUrl?. Updates question with answer, sets status to ANSWERED. Triggers AI impact assessment by calling `executeTask(questionAnsweringTask, ...)` in background (fire-and-forget via Inngest or async). Store impact assessment result on question record (impactAssessment JSON field). Emit NOTIFICATION_SEND for answer posted.
       - `flagForReview`: Takes questionId, reviewReason. Sets needsReview=true, reviewReason. Only SA role. Emit NOTIFICATION_SEND for review requested.
       - `deleteQuestion`: Soft-delete or hard-delete per schema. Confirmation handled by UI dialog per UI-SPEC destructive actions.
       - `getQuestions`: Fetch questions for project with filters (status, category, scope, owner, priority) via Prisma where clause. Support pagination. Use scopedPrisma.
       - `getQuestion`: Fetch single question with linked entities (blocking relationships, linked decisions, source transcript).
       - All actions use scopedPrisma for project isolation.

    3. Create `src/lib/agent-harness/tools/create-question.ts`:
       - Implements the `create_question` tool for the agent harness (used during transcript processing)
       - Takes: questionText, category, scope, priority, sourceTranscriptId
       - Dedup check: search existing questions for similar text (simple substring or trigram match) per D-10/TRNS-03
       - If duplicate found: return { duplicate: true, existingId, similarity } — let the AI/user decide
       - If unique: create question via Prisma, generate displayId, return new question
       - Use scopedPrisma

    4. Create `src/lib/agent-harness/tools/answer-question.ts`:
       - Implements `answer_question` tool
       - Updates question with AI-generated answer and source attribution

    5. Create `src/lib/agent-harness/tools/flag-conflict.ts`:
       - Implements `flag_conflict` tool (KNOW-07)
       - Creates a conflict record or annotation on the relevant entity when AI detects contradicting information

    6. Update `src/lib/agent-harness/tool-executor.ts`: Replace placeholder cases for create_question, answer_question, flag_conflict with imports of the real tool implementations.
