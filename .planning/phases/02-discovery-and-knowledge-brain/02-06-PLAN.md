  <action>
    1. Create `src/lib/agent-harness/tasks/article-synthesis.ts`:
       - Export `articleSynthesisTask: TaskDefinition`
       - taskType: "ARTICLE_SYNTHESIS" (add to TaskType union in types.ts)
       - executionMode: "SINGLE_TURN"
       - ambiguityMode: "BACKGROUND"
       - contextLoader: Load the existing article content + all source entities that changed since last refresh + project summary
       - tools: none (pure text generation — AI produces updated article content)
       - outputValidator: Verify output is non-empty and contains article structure
       - System prompt instructs AI to synthesize understanding from sources, update the article, note what changed

    2. Create `src/actions/knowledge.ts` using next-safe-action:
       - `getArticles(projectId)`: Fetch KnowledgeArticle records with staleness fields (isStale, staleSince, lastRefreshedAt). Include source count (how many questions/decisions/transcripts feed this article). Use scopedPrisma.
       - `getArticle(articleId, projectId)`: Fetch single article with:
         - Full content (markdown)
         - Version history (previous versions from KnowledgeArticleVersion if exists, or derive from version field)
         - Source references: linked questions, decisions, transcripts that informed the article
         - Related entities: linked business processes, features
       - `getBusinessProcesses(projectId)`: Fetch BusinessProcess records with their BusinessProcessComponent relationships (KNOW-01). Return structured list showing which Salesforce components are involved in each process.

    3. Create `src/lib/inngest/functions/staleness-detection.ts` per D-18, KNOW-03:
       - event: EVENTS.ENTITY_CONTENT_CHANGED
       - id: "staleness-detection"
       - When triggered (a question was answered, decision created, etc.):
         a. `step.run("find-affected-articles")`: Query which KnowledgeArticles reference the changed entity (via source relationships)
         b. `step.run("flag-stale")`: For each affected article, set isStale=true, staleReason=changed entity description, staleSince=now
         c. `step.sendEvent("trigger-refresh")`: Emit ARTICLE_FLAGGED_STALE event for each stale article

    4. Create `src/lib/inngest/functions/article-refresh.ts` per D-18, KNOW-04:
       - event: EVENTS.ARTICLE_FLAGGED_STALE
       - id: "article-refresh"
       - concurrency: { limit: 2, scope: "env", key: "event.data.projectId" }
       - retries: 2
       - Steps:
         a. `step.run("load-article")`: Load current article content and its sources
         b. `step.run("synthesize")`: Call `executeTask(articleSynthesisTask, { entityId: articleId }, projectId, 'system')` to generate updated content
         c. `step.run("update-article")`: Write new content, increment version, clear staleness flags (isStale=false, lastRefreshedAt=now)
         d. `step.sendEvent("trigger-embeddings")`: Emit EMBEDDING_BATCH_REQUESTED for the updated article

    5. Register both functions in `src/app/api/inngest/route.ts`
  </action>

  </verify>
  <done>Knowledge actions return articles with staleness data, staleness detection triggers on entity changes, article refresh synthesizes via agent harness and clears staleness</done>
