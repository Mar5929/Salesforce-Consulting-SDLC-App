  <action>
    1. Create `src/actions/notifications.ts` using next-safe-action:
       - `getNotifications(projectId, limit = 30)`: Fetch notifications for current user in project, ordered by priority DESC then createdAt DESC (D-30). Include read/unread status. Return last 30.
       - `getUnreadCount(projectId)`: Count unread notifications for current user. Lightweight query for SWR polling.
       - `markRead(notificationId)`: Set isRead=true, readAt=now for single notification.
       - `markAllRead(projectId)`: Update all unread notifications for current user in project to isRead=true. Non-destructive per UI-SPEC (notifications not deleted).
       - All actions verify the notification belongs to the current user (recipient check).

    2. Create `src/lib/inngest/functions/notification-dispatch.ts` per D-29, NOTF-03:
       - event: EVENTS.NOTIFICATION_SEND
       - id: "notification-dispatch"
       - concurrency: { limit: 10, scope: "env", key: "event.data.projectId" }
       - Event data: { projectId, type (NotificationType enum), title, body?, entityType, entityId, actorId (who triggered it), priority? }
       - Steps:
         a. `step.run("resolve-recipients")`: Determine who should receive this notification based on:
            - Event type: question_assigned -> assignee only. answer_posted -> question creator + watchers. review_requested -> SA role members. transcript_processed -> uploader. etc.
            - Exclude the actor (don't notify yourself)
            - Query ProjectMember records for the project
         b. `step.run("create-notifications")`: Batch create Notification records via `prisma.notification.createMany()` per tech spec pattern:
            ```
            data: recipients.map(recipientId => ({
              projectId, recipientId, type, title, body,
              entityType, entityId, priority: priority ?? 'NORMAL'
            }))
            ```
       - Handle all 10 event types per UI-SPEC Notification Event Type Copy:
         1. Question assigned: "[User] assigned you a question: [title]"
         2. Answer posted: "[User] answered: [question title]"
         3. Review requested: "[User] flagged a question for review: [title]"
         4. Transcript processed: "Transcript processing complete: [count] items extracted"
         5. Knowledge article refreshed: "Knowledge article updated: [title]"
         6. Discovery gap identified: "AI identified a discovery gap: [summary]"
         7. Contradiction detected: "Potential contradiction found: [summary]"
         8. Question unblocked: "Question unblocked: [title]"
         9. Health score changed: "Project health score changed: [old] to [new]"
         10. Sprint conflict: Reserved for Phase 3

    3. Register function in `src/app/api/inngest/route.ts`
  </action>
