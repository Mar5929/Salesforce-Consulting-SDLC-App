model TestCase {
  id             String     @id @default(cuid())
  storyId        String
  title          String
  steps          String?
  expectedResult String
  testType       TestType
  source         TestSource
  sortOrder      Int
  story          Story      @relation(...)
  testExecutions TestExecution[]
  defects        Defect[]
}

model TestExecution {
  id           String        @id @default(cuid())
  testCaseId   String
  executedById String
  executedAt   DateTime
  result       TestResult
  notes        String?
  defectId     String?
  environment  String?
  testCase     TestCase      @relation(...)
  executedBy   ProjectMember @relation(...)
  defect       Defect?       @relation(...)
}

model Defect {
  id               String        @id @default(cuid())
  projectId        String
  storyId          String?
  testCaseId       String?
  displayId        String
  title            String
  severity         DefectSeverity
  stepsToReproduce String
  expectedBehavior String
  actualBehavior   String
  environment      String?
  status           DefectStatus  @default(OPEN)
  assigneeId       String?
  duplicateOfId    String?
  createdById      String
  project          Project       @relation(...)
  story            Story?        @relation(...)
  testCase         TestCase?     @relation(...)
  assignee         ProjectMember? @relation("DefectAssignee", ...)
  createdBy        ProjectMember  @relation("DefectCreator", ...)
}

Create `src/actions/test-executions.ts` following the established server action pattern:

1. **createTestCase** action:
   - Zod input schema: `{ storyId: z.string(), title: z.string().min(1).max(500), steps: z.string().optional(), expectedResult: z.string().min(1), testType: z.enum(["HAPPY_PATH", "EDGE_CASE", "NEGATIVE", "BULK"]) }`
   - Look up story to get projectId, verify membership via `prisma.projectMember.findFirst({ where: { projectId, clerkUserId: ctx.userId } })`
   - Auto-set `source: "MANUAL"`, calculate `sortOrder` as max existing sortOrder + 1 for that story
   - Return created test case

2. **recordTestExecution** action:
   - Zod input: `{ testCaseId: z.string(), result: z.enum(["PASS", "FAIL", "BLOCKED"]), notes: z.string().optional(), environment: z.string().optional() }`
   - Look up testCase -> story -> projectId for membership check
   - Get `executedById` from the ProjectMember matching ctx.userId + projectId
   - Set `executedAt: new Date()`
   - Create TestExecution record
   - Send Inngest event `EVENTS.TEST_EXECUTION_RECORDED` with `{ projectId, storyId, testCaseId, result }`
   - Also send `EVENTS.NOTIFICATION_SEND` with type `"TEST_EXECUTION_RECORDED"` for story assignee notification
   - Return created execution

3. **getStoryTestCases** action:
   - Zod input: `{ storyId: z.string() }`
   - Returns all test cases for the story with their latest test execution (include `testExecutions: { orderBy: { executedAt: "desc" }, take: 1, include: { executedBy: true } }`)
   - Ordered by sortOrder ascending

4. **updateTestCase** action:
   - Zod input: `{ id: z.string(), title: z.string().min(1).max(500).optional(), steps: z.string().optional(), expectedResult: z.string().optional() }`
   - Update only provided fields

5. **deleteTestCase** action:
   - Zod input: `{ id: z.string() }`
   - Cascade deletes test executions (handled by Prisma onDelete: Cascade)
  </action>

   **createDefect** action:
   - Zod input: `{ projectId: z.string(), title: z.string().min(1).max(500), severity: z.enum(["LOW", "MEDIUM", "HIGH", "CRITICAL"]), stepsToReproduce: z.string().min(1), expectedBehavior: z.string().min(1), actualBehavior: z.string().min(1), storyId: z.string().optional(), testCaseId: z.string().optional(), assigneeId: z.string().optional(), environment: z.string().optional() }`
   - Verify membership via ProjectMember lookup
   - Generate displayId via `generateDisplayId(prisma, projectId, "Defect")` -- produces DEF-1, DEF-2, etc.
   - Set `createdById` from member lookup
   - Create defect record
   - Send Inngest event `EVENTS.DEFECT_CREATED` with `{ projectId, defectId, severity, storyId }`
   - Send `EVENTS.NOTIFICATION_SEND` with type `"DEFECT_CREATED"`, including assignee as recipient if provided
   - Return created defect with displayId

   **updateDefect** action:
   - Zod input: `{ id: z.string(), title: z.string().optional(), severity: z.enum([...]).optional(), stepsToReproduce: z.string().optional(), expectedBehavior: z.string().optional(), actualBehavior: z.string().optional(), assigneeId: z.string().nullable().optional(), environment: z.string().optional() }`
   - Verify membership and ownership or PM/SA role
   - Update only provided fields
   - If assigneeId changes, send notification to new assignee

   **transitionDefectStatus** action:
   - Zod input: `{ id: z.string(), toStatus: z.enum(["OPEN", "ASSIGNED", "FIXED", "VERIFIED", "CLOSED"]) }`
   - Load current defect, get member role
   - Call `canTransitionDefect(defect.status, toStatus, member.role)` -- throw if false
   - Update status
   - Send `EVENTS.DEFECT_STATUS_CHANGED` with `{ projectId, defectId, fromStatus, toStatus }`
   - Send `EVENTS.NOTIFICATION_SEND` with type `"DEFECT_STATUS_CHANGED"`
   - Return updated defect

   **getDefects** action:
   - Zod input: `{ projectId: z.string(), status: z.enum([...]).optional(), severity: z.enum([...]).optional(), assigneeId: z.string().optional(), storyId: z.string().optional() }`
   - Query with optional filters, include story (select id, displayId, title), assignee (select id, displayName), createdBy (select id, displayName)
   - Order by createdAt desc

   **getDefect** action:
   - Zod input: `{ id: z.string() }`
   - Return full defect with relations: story, testCase, assignee, createdBy, duplicateOf

   **deleteDefect** action:
   - Zod input: `{ id: z.string() }`
   - Verify membership and ownership or PM/SA role
   - Delete defect record
