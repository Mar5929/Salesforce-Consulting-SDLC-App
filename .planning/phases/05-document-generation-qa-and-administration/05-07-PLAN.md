3. Create `src/lib/jira/field-mapping.ts`:
   - Map app fields to Jira fields per D-16:
   ```typescript
   import type { StoryStatus } from "@/generated/prisma"

   export const STATUS_TO_JIRA: Record<StoryStatus, string> = {
     DRAFT: "To Do",
     READY: "To Do",
     SPRINT_PLANNED: "To Do",
     IN_PROGRESS: "In Progress",
     IN_REVIEW: "In Review",
     QA: "In Review",
     DONE: "Done",
   }

   export function mapStoryToJiraFields(story: {
     title: string
     description: string | null
     priority: string | null
     storyPoints: number | null
     displayId: string
   }) {
     return {
       summary: `[${story.displayId}] ${story.title}`,
       description: story.description ?? "",
       // priority and story_points mapped if Jira project supports them
       ...(story.storyPoints ? { customfield_story_points: story.storyPoints } : {}),
     }
   }
   ```

4. Create `src/lib/jira/sync.ts`:
   - Import `createJiraClient` and `mapStoryToJiraFields`
   - Export `async function pushStoryToJira(config: JiraConfig, story: StoryWithDetails): Promise<{ jiraIssueId: string; jiraIssueKey: string }>`:
     - Create Jira client
     - Check if JiraSyncRecord exists for this story
     - If no existing Jira issue: create issue via `client.issues.createIssue({ fields: { project: { key: config.jiraProjectKey }, issuetype: { name: "Story" }, ...mapStoryToJiraFields(story) } })`
     - If existing: update issue via `client.issues.editIssue({ issueIdOrKey, fields: ... })`
     - Return jiraIssueId and jiraIssueKey
   - Export `async function syncStoryStatus(config: JiraConfig, storyId: string, newStatus: StoryStatus): Promise<void>`:
     - Look up JiraSyncRecord for storyId to get jiraIssueKey
     - If no record, skip (story not synced yet)
     - Get available transitions: `client.issues.getTransitions({ issueIdOrKey })`
     - Find transition matching `STATUS_TO_JIRA[newStatus]`
     - If found: `client.issues.doTransition({ issueIdOrKey, transition: { id } })`
     - If not found: log warning (Jira workflow may not match)

1. Create `src/lib/inngest/functions/jira-sync.ts`:
   - Inngest function triggered by `EVENTS.STORY_STATUS_CHANGED` (existing Phase 3 event)
   - ID: "jira-sync-on-status-change", retries: 2
   - Steps:
     1. Check if project has JiraConfig with enabled=true. If not, return early.
     2. Load story with relations
     3. Call `pushStoryToJira` to create/update the issue in Jira
     4. Call `syncStoryStatus` to transition the Jira issue status
     5. Upsert JiraSyncRecord: `{ projectId, storyId, jiraIssueId, jiraIssueKey, status: "SYNCED", lastSyncAt: new Date() }`
     6. On error: upsert JiraSyncRecord with `{ status: "FAILED", errorMessage: error.message }`
     7. Send `EVENTS.NOTIFICATION_SEND` for sync completion/failure
   - Register in `src/app/api/inngest/route.ts`

2. Create `src/actions/jira-sync.ts`:

   **saveJiraConfig** action:
   - Zod input: `{ projectId: z.string(), instanceUrl: z.string().url(), email: z.string().email(), apiToken: z.string().min(1), jiraProjectKey: z.string().min(1).max(10) }`
   - Verify PM or SA role
   - Encrypt apiToken: `const encryptedToken = await encrypt(apiToken, projectId)`
   - Upsert JiraConfig: create or update for the project
   - Return config (without decrypted token)

   **getJiraConfig** action:
   - Zod input: `{ projectId: z.string() }`
   - Return JiraConfig without encryptedToken (select all fields except token, or mask it)

   **toggleJiraSync** action:
   - Zod input: `{ projectId: z.string(), enabled: z.boolean() }`
   - Update JiraConfig.enabled

   **getJiraSyncStatus** action:
   - Zod input: `{ projectId: z.string(), storyIds: z.array(z.string()).optional() }`
   - Return JiraSyncRecord[] for the project (optionally filtered by storyIds)
   - Include status, lastSyncAt, jiraIssueKey, errorMessage

   **retryFailedSyncs** action:
   - Zod input: `{ projectId: z.string() }`
   - Find all JiraSyncRecords with status "FAILED"
   - For each, send `EVENTS.JIRA_SYNC_REQUESTED` with storyId
   - Return count of retries initiated

3. Create `src/actions/project-archive.ts`:

   **archiveProject** action:
   - Zod input: `{ projectId: z.string() }`
   - Verify PM role
   - Check no active sprints (sprints with status IN_PROGRESS). If found, throw: "Active sprints must be completed or cancelled first."

   - Send `EVENTS.PROJECT_ARCHIVED` with `{ projectId }`
   - Send `EVENTS.NOTIFICATION_SEND` with type `"PROJECT_ARCHIVED"` to all members
   - Return updated project

   **reactivateProject** action:
   - Zod input: `{ projectId: z.string() }`
   - Verify PM role
   - Check project is actually ARCHIVED. If not, throw.
   - Update `project.status = "ACTIVE"`
   - Send `EVENTS.PROJECT_REACTIVATED` with `{ projectId }`
   - Send `EVENTS.NOTIFICATION_SEND` with type `"PROJECT_REACTIVATED"` to all members
   - Return updated project
  </action>
