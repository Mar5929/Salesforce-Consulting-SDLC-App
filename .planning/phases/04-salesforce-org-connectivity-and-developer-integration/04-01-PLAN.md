2. Create `src/lib/salesforce/oauth.ts` per RESEARCH Pattern 1:
- `buildAuthorizationUrl(projectId: string): string` — builds Salesforce OAuth authorize URL with `response_type=code`, `client_id` from `process.env.SF_CONNECTED_APP_CLIENT_ID`, `redirect_uri` from `process.env.NEXT_PUBLIC_APP_URL + "/api/auth/salesforce/callback"`, `state` = projectId
- `exchangeCodeForTokens(code: string, projectId: string): Promise<{ instanceUrl: string; encryptedAccessToken: string; encryptedRefreshToken: string | null }>` — creates jsforce OAuth2 and Connection, calls `conn.authorize(code)`, encrypts tokens using `encrypt()` from encryption.ts

3. Create `src/lib/salesforce/client.ts` per RESEARCH Pattern 2:
- `getSalesforceConnection(projectId: string): Promise<jsforce.Connection>` — fetches project sfOrg fields, decrypts tokens, creates jsforce.Connection with oauth2 config, registers `conn.on("refresh", ...)` handler to re-encrypt and persist new access token

4. Create `src/lib/api-keys/generate.ts`:
- `generateApiKey(projectId: string): Promise<{ rawKey: string; keyPrefix: string; keyHash: string }>` — generates key as `sfai_${projectId.slice(0,8)}_${32 random hex chars}`, stores first 8 chars as keyPrefix, bcrypt hashes full key with cost factor 12

5. Create `src/lib/api-keys/middleware.ts` per RESEARCH Pattern 4:
- `validateApiKey(request: Request): Promise<{ projectId: string; apiKeyId: string } | NextResponse>` — reads `x-api-key` header, returns 401 if missing. Extracts first 8 chars as prefix, queries `prisma.apiKey.findMany({ where: { keyPrefix, isActive: true } })`, bcrypt compares each match. On match: updates lastUsedAt and useCount, returns projectId+apiKeyId. On no match: returns 401.

6. Create `src/lib/api-keys/rate-limit.ts` per RESEARCH Pattern 5:
- `checkRateLimit(apiKeyId: string, endpoint: string, limit: number = 60, windowMs: number = 60_000): Promise<{ allowed: boolean; remaining: number; retryAfter?: number }>` — counts ApiRequestLog entries within sliding window, if under limit logs request and returns allowed=true, else returns allowed=false with retryAfter

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Salesforce OAuth callback | External redirect from Salesforce login page carries auth code |
| /api/v1/ endpoints | External API consumers (Claude Code) authenticate via API key |
| Token storage | Encrypted SF tokens stored in Postgres, decrypted only in-memory |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Information Disclosure | SF token storage | mitigate | Encrypt tokens with HKDF-SHA256 per-project key via encryption.ts. Never expose in API responses. |
| T-04-02 | Tampering | API key brute-force | mitigate | bcrypt hash (cost 12), keyPrefix index for fast rejection, Postgres rate limiter (60 req/min default) |
| T-04-03 | Elevation of Privilege | API key cross-project | mitigate | API keys scoped to single project, all queries use project-scoped Prisma |
| T-04-04 | Information Disclosure | API key enumeration | mitigate | bcrypt constant-time compare, generic "Invalid API key" error message |
</threat_model>
