## Typography

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Metadata | 13px | 400 (regular) | 1.4 | Timestamps, sync history entries, component counts, last-synced dates, API key creation dates |
| Label | 13px | 600 (semibold) | 1.2 | Form labels, settings section headers, column headers, badge text, tab labels |
| Body | 14px | 400 (regular) | 1.5 | Descriptions, table cells, review card content, domain grouping descriptions, component names |

Notes:
- Inherited typography system unchanged from Phases 1-2. No new sizes or weights introduced.
- Monospace exception for API key display only: `"SF Mono", "Fira Code", "Fira Mono", Menlo, monospace` at 13px/400.
- Confidence percentages on review cards use 13px/600 label style.

## Color

| Role | Value | CSS Variable | Usage |
|------|-------|-------------|-------|
| Dominant (60%) | hsl(0, 0%, 100%) / #FFFFFF | `--background` | Page backgrounds, content surfaces |
| Secondary (30%) | hsl(0, 0%, 98%) / #FAFAFA | `--muted` | Cards, sidebar, table headers, settings page sections |
| Accent (10%) | hsl(220, 70%, 50%) / #2563EB | `--primary` | See reserved-for list below |
| Destructive | hsl(0, 84%, 60%) / #EF4444 | `--destructive` | Destructive actions, connection error states |

### Secondary Palette (inherited from Phase 1)

| Role | Value | CSS Variable | Usage |
|------|-------|-------------|-------|
| Border | hsl(0, 0%, 90%) / #E5E5E5 | `--border` | Card borders, table dividers, input borders, settings section borders |
| Muted text | hsl(0, 0%, 45%) / #737373 | `--muted-foreground` | Timestamps, metadata, placeholder text, component counts |
| Text primary | hsl(0, 0%, 9%) / #171717 | `--foreground` | Headings, body text, component names, domain names |
| Text secondary | hsl(0, 0%, 32%) / #525252 | `--secondary-foreground` | Descriptions, helper text, sync history details |

### Semantic Colors (Phase 4 additions)

| Role | Value | Usage |
|------|-------|-------|
| Connected | #22C55E (green-500) | Org connection status badge when authenticated and healthy |
| Warning | #F59E0B (amber-500) | "Needs Reauthorization" connection status, low-confidence AI suggestion indicator |
| Error | #EF4444 (reuse destructive) | Connection failed, sync failed, token expired states |

These semantic colors are used exclusively for connection health and confidence indicators. Connected (#22C55E) pairs with the success semantic established in Phase 2. Warning (#F59E0B) pairs with the aging semantic from Phase 2. Do not use these colors for buttons, links, or decorative purposes.

### Accent Reserved For (Phase 4 additions to existing list)

- Primary CTA buttons ("Connect Salesforce Org", "Analyze Org", "Generate API Key")
- Active sidebar navigation item (text color + left border indicator)
- Focus rings on interactive elements
- Link text on hover
- "Confirm" action on accept/reject review cards
- Active tab indicator in brownfield review
- Bulk-confirm button for high-confidence AI suggestions

Accent is NOT used for: connection status badges (use semantic colors), confidence indicators, component type labels, sync status indicators, or decorative purposes.

### Connection Status Badge Colors

| Status | Style |
|--------|-------|
| Connected | Solid badge, #22C55E background, white text, dot indicator |
| Disconnected | Outline badge, `--muted-foreground` text |
| Needs Reauthorization | Solid badge, #F59E0B background, white text |
| Syncing | Outline badge, `--primary` text, animated pulse dot |
| Sync Failed | Solid badge, `--destructive` background, white text |

### AI Suggestion Confidence Badges

| Confidence | Style |
|------------|-------|
| High (80%+) | Solid badge, #22C55E background, white text |
| Medium (50-79%) | Solid badge, #F59E0B background, white text |
| Low (<50%) | Outline badge, `--muted-foreground` text |

## Layout Contract

### Org Connection Settings Card (D-01, D-08)

```
+--[ Settings Page ]--------------------------------------------+
|                                                                 |
|  Org Connection                                    18px/600     |
|  ─────────────────────────────────────────────────────          |
|  +--[ Status Card ]---------------------------------------+    |
|  |  [Cloud icon]  Salesforce Org              [Badge: ●]  |    |
|  |                                           Connected     |    |
|  |  Instance: acme.my.salesforce.com                       |    |
|  |  Last synced: 2 hours ago  |  Components: 847          |    |
|  |                                                         |    |
|  |  [ Sync Now ]  [ Disconnect ]                           |    |
|  +--------------------------------------------------------+    |
|                                                                 |
|  Sync Schedule                                                  |
|  ─────────────────────────────────────────────────────          |
|  Incremental sync   [Toggle: ON]   Every 24 hours              |
|                                                                 |
|  Sync History                                                   |
|  ─────────────────────────────────────────────────────          |
|  | Date       | Type        | Added | Modified | Removed |     |
|  | 2026-04-06 | Incremental | 3     | 12       | 0       |     |
|  | 2026-04-05 | Full        | 847   | 0        | 0       |     |
+-----------------------------------------------------------------+
```

- Status card: Card component, 24px padding, border-left 3px colored by connection status
- Sync Now: outline button. Disconnect: ghost button with destructive text on hover.
- Sync history: Table component, 5 most recent entries, "View All" link if more exist.

### Org Analysis Page (D-10, D-14)

```
+--[ Org Analysis ]-----------------------------------------+
|                                                             |
|  Org Analysis                                24px/600      |
|                                                             |
|  +--[ Pipeline Status ]-------------------------------+    |
|  |  [1 Parse ●]---[2 Classify ●]---[3 Synthesize ◐]  |    |
|  |                                  In progress...     |    |
|  +----------------------------------------------------+    |
|                                                             |
|  +--[ Tabs ]----------------------------------------------+|
|  | [Domain Groupings]  [Business Processes]               ||
|  |---------------------------------------------------------||
|  |                                                         ||
|  | [ Confirm All High-Confidence ] (accent)    [3 of 12]  ||
|  |                                                         ||
|  | +--[ Review Card ]----------------------------------+   ||
|  | | [Domain Name]                    [Badge: 92%]     |   ||
|  | | AI suggested grouping of 14 components            |   ||
|  | |                                                   |   ||
|  | | Components: Account, Contact, Opportunity, ...    |   ||
|  | |                                                   |   ||
|  | | [Confirm]  [Edit]  [Reject]                       |   ||
|  | +--------------------------------------------------+   ||
|  |                                                         ||
|  | +--[ Review Card ]----------------------------------+   ||
|  | | [Domain Name]                    [Badge: 61%]     |   ||
|  | | ...                                               |   ||
|  | +--------------------------------------------------+   ||
|  +---------------------------------------------------------+|
+-------------------------------------------------------------+
```

- Pipeline status: horizontal stepper with numbered circles, connected by lines. Completed steps: accent fill. Active: accent outline with pulse animation. Upcoming: muted border.
- Tabs: standard shadcn Tabs, underline indicator style.
- Review cards: Card component, 24px padding. Confidence badge top-right. Accept/reject/edit pattern reused from Phase 2 transcript extraction review (D-12).
- Confirm button: accent outline. Edit button: ghost. Reject button: ghost with destructive text on hover.
- "Confirm All High-Confidence": accent button, only enabled when unconfirmed high-confidence items exist.
- Counter "[3 of 12]" shows confirmed/total in 13px/400 muted text.

### API Key Management (D-15)

```
+--[ Developer API Settings ]-------------------------------+
|                                                             |
|  API Keys                                     18px/600     |
|  ─────────────────────────────────────────────────────      |
|  [ Generate API Key ] (accent)                              |
|                                                             |
|  | Name          | Created    | Last Used  | Actions |     |
|  | claude-code-1 | 2026-04-05 | 2 hrs ago  | [Revoke]|     |
|  | ci-pipeline   | 2026-04-01 | Never      | [Revoke]|     |
|                                                             |
|  API Documentation                                          |
|  ─────────────────────────────────────────────────────      |
|  Base URL: https://app.example.com/api/v1                   |
|  Auth: x-api-key header                                     |
|  Endpoints: [collapsible list]                              |
+-------------------------------------------------------------+
```

- API key creation flow: Dialog with name input. On create, show the key ONCE in a monospace card with copy-to-clipboard button. Warning text: "Copy this key now. It will not be shown again."
- API key shown once: Card with `--muted` background, monospace 13px font, Copy icon button. Card auto-dismisses after user confirms copy or navigates away.
- Key table: Table component. "Revoke" is ghost button, destructive text color.
- Endpoint documentation: Collapsible sections per endpoint showing method, path, description. Code-style formatting for paths.

### Loading States

- Org connection card: Skeleton matching card layout (icon placeholder, text lines, button placeholders)
- Org component table: Table skeleton with 5 rows of shimmering cells
- Pipeline status: Stepper skeleton with 3 circles and connecting lines
- Review cards: 3 card skeletons with title line, body lines, and button row
- API key table: Table skeleton with 2 rows

### Error Boundaries

- OAuth flow error: Alert component (destructive variant) at top of org settings page. Dismissible.
- Sync failure: Alert component (destructive variant) below connection status card. Includes "Retry" action.
- Ingestion failure: Pipeline stepper shows failed step in destructive color. Alert below stepper with error details and "Retry" action.
- API section error: Inline card with error message and retry.
