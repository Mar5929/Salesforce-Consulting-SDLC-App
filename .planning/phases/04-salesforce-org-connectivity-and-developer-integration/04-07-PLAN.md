1. Create `docs/claude-code-skills/context-package.md`:
   - Title: "Context Package Retrieval"
   - Description: "Fetch full context for a story before starting development work. Includes story details, related business processes, relevant knowledge articles, project decisions, and sprint conflict warnings."
   - Prerequisites: "Set SFAI_API_KEY environment variable (generated from project Settings > Developer API)"
   - Base URL: "Set SFAI_BASE_URL environment variable (e.g., https://your-app.vercel.app)"
   - Usage example with curl:
     ```bash
     curl -s -H "x-api-key: $SFAI_API_KEY" \
       "$SFAI_BASE_URL/api/v1/context-package?storyId=STORY_ID"
     ```
   - Response shape documentation with field descriptions
   - When to use: "Before starting work on any story. The context package provides the business context, related org components, and any sprint conflicts the developer should be aware of."
   - Error handling: 401 = invalid key, 404 = story not found, 429 = rate limited

2. Create `docs/claude-code-skills/org-query.md`:
   - Title: "Org Metadata Query"
   - Description: "Query Salesforce org metadata to understand existing customizations when implementing stories. Filter by component type or domain grouping."
   - Same env var setup as above
   - Usage examples:
     ```bash
     # List all custom objects
     curl -s -H "x-api-key: $SFAI_API_KEY" \
       "$SFAI_BASE_URL/api/v1/org/components?type=OBJECT"

     # List all Apex classes in a domain
     curl -s -H "x-api-key: $SFAI_API_KEY" \
       "$SFAI_BASE_URL/api/v1/org/components?type=APEX_CLASS&domain=DOMAIN_ID"

     # Paginate results
     curl -s -H "x-api-key: $SFAI_API_KEY" \
       "$SFAI_BASE_URL/api/v1/org/components?page=2&pageSize=50"
     ```
   - Response shape with pagination fields
   - Valid type values: OBJECT, FIELD, APEX_CLASS, APEX_TRIGGER, LWC, FLOW, PERMISSION_SET, PROFILE, OTHER
   - When to use: "When you need to understand the existing Salesforce org structure — what objects exist, what automations are in place, what the current state of a component is."

3. Create `docs/claude-code-skills/story-status.md`:
   - Title: "Story Status Update"
   - Description: "Update a story's execution status as you progress through development. Valid transitions: SPRINT_PLANNED -> IN_PROGRESS, IN_PROGRESS -> IN_REVIEW, IN_REVIEW -> QA."
   - Usage:
     ```bash
     curl -s -X PATCH \
       -H "x-api-key: $SFAI_API_KEY" \
       -H "Content-Type: application/json" \
       -d '{"status": "IN_PROGRESS"}' \
       "$SFAI_BASE_URL/api/v1/stories/STORY_ID/status"
     ```
   - Valid status values: DRAFT, READY, SPRINT_PLANNED, IN_PROGRESS, IN_REVIEW, QA, DONE
   - Valid developer transitions: SPRINT_PLANNED->IN_PROGRESS, IN_PROGRESS->IN_REVIEW
   - When to use: "When starting work on a story (set IN_PROGRESS), when submitting for review (set IN_REVIEW). The web app will notify the PM and update the sprint board."

4. Create `docs/claude-code-skills/project-init.md`:
   - Title: "Project Session Initialization"
   - Description: "Fetch project summary at the start of a Claude Code session to understand the project context, team size, and current state."
   - Usage:
     ```bash
     curl -s -H "x-api-key: $SFAI_API_KEY" \
       "$SFAI_BASE_URL/api/v1/project/summary"
     ```
   - Response shape with all fields documented
   - When to use: "At the start of every Claude Code session. Gives you the project name, client, number of epics/stories, open questions, and team composition."
