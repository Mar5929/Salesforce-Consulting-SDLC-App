# BUG-041: Kanban board drag-and-drop doesn't work for features/epics

- **Severity:** MEDIUM
- **Phase:** 3 (Story Management and Sprint Intelligence)
- **Status:** Fixed

## Description

In the Work tab, when viewing features in Kanban mode (inside an epic), cards cannot be dragged to different status columns. The same applies to the epic-level Kanban view. The user expects consistent drag-and-drop behavior across all Kanban boards in the app, but only some boards support it.

## Location

- `src/components/work/feature-kanban.tsx:1-111` — Feature kanban (NO drag-and-drop)
- `src/components/work/epic-kanban.tsx:1-113` — Epic kanban (NO drag-and-drop)
- `src/components/work/story-kanban.tsx:1-205` — Story kanban (HAS drag-and-drop)
- `src/components/questions/question-kanban.tsx:1-209` — Question kanban (HAS drag-and-drop)

## Root Cause

The Feature and Epic Kanban components were intentionally built as navigation-only boards — clicking a card navigates to the detail page. They do not implement any drag-and-drop handlers (no `onDragStart`, `onDragOver`, `onDrop` events).

In contrast, Story Kanban and Question Kanban use HTML5 drag API with `onDragStart`, `onDragOver`, `onDrop` handlers and call server actions to update status on drop.

This inconsistency is confusing — users expect all Kanban boards to support drag-and-drop for status changes.

## Fix

1. Add HTML5 drag-and-drop to `feature-kanban.tsx` following the same pattern as `story-kanban.tsx`
2. Create/use an `updateFeatureStatus` server action for the drop handler
3. Add HTML5 drag-and-drop to `epic-kanban.tsx` similarly
4. Create/use an `updateEpicStatus` server action for the drop handler
5. Consider extracting a shared `DraggableKanban` component to avoid duplicating drag logic across 4+ kanban boards
