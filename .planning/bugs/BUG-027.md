# BUG-027: bulkConfirmHighConfidence confirms ALL records

- **Severity:** LOW
- **Phase:** 4 (Salesforce Org Connectivity)
- **Status:** Open

## Description

The `bulkConfirmHighConfidence` server action is supposed to confirm only high-confidence AI suggestions, but it actually confirms ALL unconfirmed domain groupings or business processes for a project. The button label says "Confirm All High-Confidence" but the action has no confidence-based filtering. Users who click this button expecting to confirm only strong AI suggestions will inadvertently confirm every unconfirmed record, including low-confidence ones that need human review.

## Location

- `src/actions/org-analysis.ts:280-317` — `bulkConfirmHighConfidence` function confirms all unconfirmed records without confidence filtering

## Root Cause

The `DomainGrouping` and `BusinessProcess` Prisma models do not have a `confidence` field. When the ingestion pipeline classifies components (in `src/lib/inngest/functions/org-ingestion.ts:137`), the AI returns a numeric `confidence` percentage per grouping, but this value is only appended to the `description` string as text (e.g., `"(confidence: 85%)"`) rather than stored in a dedicated database column. As a result, the `bulkConfirmHighConfidence` action cannot filter by confidence and simply confirms everything with `isConfirmed: false`.

The code comment on lines 278-279 explicitly acknowledges this:
> "Note: Confidence filtering is not supported because DomainGrouping and BusinessProcess models do not store a numeric confidence score."

## Fix

1. **Add a `confidence` column** to both `DomainGrouping` and `BusinessProcess` models in `prisma/schema.prisma`:
   ```prisma
   model DomainGrouping {
     ...
     confidence     Float?   // AI confidence score (0-1)
   }
   
   model BusinessProcess {
     ...
     confidence     Float?   // AI confidence score (0-1)
   }
   ```

2. **Store the confidence score** in `src/lib/inngest/functions/org-ingestion.ts`:
   - In `classifyComponents` (~line 133): Store `grouping.confidence / 100` in the `confidence` field instead of embedding it in the description string.
   - In `synthesizeBusinessProcesses`: If the synthesize task returns a confidence score, store it similarly.

3. **Filter by confidence** in `bulkConfirmHighConfidence` (`src/actions/org-analysis.ts`):
   ```ts
   const HIGH_CONFIDENCE_THRESHOLD = 0.8

   // For domain groupings:
   where: {
     projectId,
     isConfirmed: false,
     confidence: { gte: HIGH_CONFIDENCE_THRESHOLD },
   }
   ```

4. **Clean up description text**: Remove the `(confidence: XX%)` suffix from the description string in the ingestion function, since the value will be stored in a proper column.
