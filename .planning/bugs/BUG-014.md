# BUG-014: updateStoryStatus doesn't fire PROJECT_STATE_CHANGED

- **Severity:** HIGH
- **Phase:** 8 (Event Wiring and Integration Fixes)
- **Status:** Open

## Description

When a story's status is changed via `updateStoryStatus`, the action fires `STORY_STATUS_CHANGED` and `NOTIFICATION_SEND` events but does not fire the `PROJECT_STATE_CHANGED` event. The dashboard synthesis Inngest function is triggered exclusively by `PROJECT_STATE_CHANGED` events, so story status transitions (e.g., moving a story from IN_PROGRESS to DONE) never cause the dashboard metrics or AI focus summary to update. This means the PM dashboard becomes stale and out of sync with actual project progress after story status changes, which are among the most frequent state-changing operations in the app.

## Location

- `src/actions/stories.ts:304-314` Б─■ Fires `STORY_STATUS_CHANGED` but not `PROJECT_STATE_CHANGED`

## Root Cause

The `updateStoryStatus` action was wired to emit `STORY_STATUS_CHANGED` (for sprint-level processing) and `NOTIFICATION_SEND` (for assignee notification), but the `PROJECT_STATE_CHANGED` event Б─■ which triggers dashboard re-synthesis Б─■ was never added. Other actions like `answerQuestion`, `updateQuestionStatus`, and `resolveQuestion` in `src/actions/questions.ts` correctly fire `PROJECT_STATE_CHANGED` after mutations, and `transcript-processing.ts` and `article-refresh.ts` do as well. This was simply an omission during Phase 3 or Phase 7 wiring.

## Fix

In `src/actions/stories.ts`, add a `PROJECT_STATE_CHANGED` event send after the existing `STORY_STATUS_CHANGED` send (around line 314). Insert:

```ts
await inngest.send({
  name: EVENTS.PROJECT_STATE_CHANGED,
  data: {
    projectId: parsedInput.projectId,
  },
})
```

This should be placed after the `STORY_STATUS_CHANGED` send at line 304-314, before the notification send. The `EVENTS` import and `inngest` client are already imported in this file.

Additionally, consider whether `createStory`, `updateStory`, and `deleteStory` in the same file should also fire `PROJECT_STATE_CHANGED`, since they also mutate project state that the dashboard should reflect.
