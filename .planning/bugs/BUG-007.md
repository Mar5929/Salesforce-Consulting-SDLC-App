# BUG-007: QuestionForm DialogTrigger renders empty button

- **Severity:** HIGH
- **Phase:** 2 (Discovery and Knowledge)
- **Status:** Fixed

## Description

When `QuestionForm` is rendered without a custom `trigger` prop (the default case on the questions page), the `DialogTrigger` renders an empty `Button` with no visible text. The `render` prop receives a `<Button>` element, and the children text "Ask Question" is passed as children of `DialogTrigger`, but the base-ui `DialogTrigger` component uses `render` to replace its root element -- the children "Ask Question" may not be composed into the rendered Button correctly. Additionally, when a custom `trigger` is passed, the code casts it as `React.ReactElement` for the `render` prop, which is fragile and may not work with all React node types.

## Location

- `src/components/questions/question-form.tsx:98-106` — DialogTrigger uses `render` prop with conditional logic that produces an empty button when no trigger is provided

## Root Cause

The `DialogTrigger` from `@base-ui/react/dialog` uses a `render` prop pattern where the rendered element replaces the trigger root. On line 100-101, when no custom `trigger` is provided, `render` receives `<Button className="bg-[#2563EB] text-white hover:bg-[#1d4ed8]" />` -- a self-closing Button with no children. The text "Ask Question" is passed as children of `DialogTrigger` (line 105), but base-ui's `render` prop replaces the entire element tree, so the children are not composed into the Button. The user sees a styled button with no label.

## Fix

1. Replace the current `DialogTrigger` block with the standard `asChild` pattern or pass children directly into the rendered Button:

```tsx
<DialogTrigger
  render={
    trigger ?? (
      <Button className="bg-[#2563EB] text-white hover:bg-[#1d4ed8]">
        Ask Question
      </Button>
    )
  }
/>
```

2. Remove the conditional `{trigger ? undefined : "Ask Question"}` children since the text should live inside the `render` element.
