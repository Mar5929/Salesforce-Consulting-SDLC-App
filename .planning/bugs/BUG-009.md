# BUG-009: Checkbox indeterminate prop silently ignored on Base UI

- **Severity:** HIGH
- **Phase:** 3 (Story Management)
- **Status:** Open

## Description

The `Checkbox` component in `src/components/ui/checkbox.tsx` wraps Base UI's `Checkbox.Root` and correctly passes the `indeterminate` prop through via `...props`. Base UI does handle this prop and will set `data-indeterminate` on the root and render the `CheckboxIndicator` when indeterminate is true. However, the Indicator only renders a `CheckIcon` (checkmark) -- there is no visual distinction between checked and indeterminate states. Users see a full checkmark when some (but not all) rows are selected in the StoryTable header, which is misleading. The expected UX is a minus/dash icon for the indeterminate state.

## Location

- `src/components/ui/checkbox.tsx:18-24` -- The `CheckboxPrimitive.Indicator` always renders a `CheckIcon` regardless of whether the state is checked or indeterminate
- `src/components/work/story-table.tsx:167` -- Header checkbox passes `indeterminate` prop, expecting visual differentiation

## Root Cause

The `Checkbox` component's indicator section unconditionally renders a `CheckIcon`. Base UI's `Checkbox.Indicator` shows its children when either `checked` or `indeterminate` is true, but the component makes no effort to switch the icon based on state. Since there is no conditional rendering of a `MinusIcon` for the indeterminate case, both states look identical to the user.

## Fix

1. In `src/components/ui/checkbox.tsx`, consume the `indeterminate` prop and conditionally render a `MinusIcon` (from `lucide-react`) when indeterminate, and a `CheckIcon` when checked. Base UI exposes state via the `data-indeterminate` attribute, but for the icon swap, the cleanest approach is to destructure `indeterminate` from props and use it to select the icon:

```tsx
import { CheckIcon, MinusIcon } from "lucide-react"

function Checkbox({ className, indeterminate, ...props }: CheckboxPrimitive.Root.Props) {
  return (
    <CheckboxPrimitive.Root
      data-slot="checkbox"
      className={cn("...", className)}
      indeterminate={indeterminate}
      {...props}
    >
      <CheckboxPrimitive.Indicator
        data-slot="checkbox-indicator"
        className="grid place-content-center text-current transition-none [&>svg]:size-3.5"
      >
        {indeterminate ? <MinusIcon /> : <CheckIcon />}
      </CheckboxPrimitive.Indicator>
    </CheckboxPrimitive.Root>
  )
}
```

2. Verify in the StoryTable that the header "select all" checkbox now displays a dash when some rows are selected.
