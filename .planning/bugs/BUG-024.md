# BUG-024: No UI to create test cases -- QA tab dead end

- **Severity:** MEDIUM
- **Phase:** 5 (Documents, QA, Admin)
- **Status:** Fixed

## Description

The QA tab on the story detail page shows a `TestExecutionTable` component that displays existing test cases and allows recording execution results (Pass/Fail/Blocked). However, there is no UI anywhere in the app to create test cases in the first place. The `createTestCase` server action exists in `src/actions/test-executions.ts` and accepts title, steps, expected result, and test type, but no component calls it. The QA tab's empty state says "No test results" and offers a "Record Result" button, which is misleading -- you cannot record results without test cases to record against. This makes the entire QA workflow a dead end.

## Location

- `src/components/qa/test-execution-table.tsx:137-155` -- Empty state offers "Record Result" but no "Create Test Case"
- `src/app/(dashboard)/projects/[projectId]/work/[epicId]/[featureId]/stories/[storyId]/story-detail-client.tsx:233-239` -- QA tab only renders TestExecutionTable

## Root Cause

The `createTestCase` server action was implemented but the corresponding UI form component was never built. The `TestExecutionTable` component only handles viewing test cases and recording results against existing ones. There is no `TestCaseCreateForm` or similar component, and the empty state does not provide a way to create the initial test cases needed to begin QA work.

## Fix

1. Create a `TestCaseCreateForm` component (e.g., `src/components/qa/test-case-create-form.tsx`) that calls the `createTestCase` action with fields for title, steps (optional), expected result, and test type (HAPPY_PATH, EDGE_CASE, NEGATIVE, BULK).

2. Add a "Create Test Case" button to the `TestExecutionTable` component header (next to the existing "Record Result" button) and to the empty state.

3. The empty state message should be updated from "No test results" to "No test cases" with guidance to create test cases first.

## Related Files

- `src/actions/test-executions.ts` -- Contains `createTestCase` action (lines 82-111), already fully implemented
- `src/components/qa/record-result-form.tsx` -- Existing form pattern to follow for the new create form
