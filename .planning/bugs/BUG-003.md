# BUG-003: S3 env vars missing from Vercel — all document generation fails silently

- **Severity:** CRITICAL
- **Phase:** 5 (Documents)
- **Status:** Fixed

## Description

Document generation requires S3/R2 storage for uploading generated files. The Inngest `generateDocumentFunction` calls `uploadDocument()` which uses the S3 client from `s3-storage.ts`. The client requires four environment variables that are set in `.env.local` for local dev but are not:
1. Documented in `.env.example`
2. Configured in Vercel's environment variables

When the Inngest function runs on Vercel, `getBucket()` throws `"AWS_S3_BUCKET environment variable is required"`, the step fails, Inngest retries twice, then the `GeneratedDocument` record is never created. The user sees the generation dialog stuck at "Generating..." forever.

## Location

- `src/lib/documents/s3-storage.ts:19-27` — S3 client initialization requiring env vars
- `src/lib/inngest/functions/document-generation.ts` — the Inngest function that calls S3 upload

## Root Cause

S3 env vars were set locally but never added to Vercel deployment configuration or documented for deployment.

## Fix

1. Add to Vercel environment variables (Production + Preview):
   - `AWS_ACCESS_KEY_ID`
   - `AWS_SECRET_ACCESS_KEY`
   - `AWS_S3_BUCKET`
   - `AWS_S3_REGION`
   - `AWS_S3_ENDPOINT` (optional, for Cloudflare R2)
2. Add all five to `.env.example` with placeholder values
3. Consider adding a startup validation check (e.g., via `@t3-oss/env-nextjs`)

## Related Files

- `archive/.env.example` — current env example (missing S3 vars)
- `src/lib/documents/renderers/` — PDF, DOCX, PPTX renderers that produce the files to upload
