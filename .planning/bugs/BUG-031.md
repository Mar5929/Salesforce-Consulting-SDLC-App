# BUG-031: Sidebar fallback paths 404 when no project active

- **Severity:** LOW
- **Phase:** CC (Claude Code)
- **Status:** Fixed

## Description

When no project is active (e.g., user is on the root `/` projects list page), the sidebar navigation items for project-scoped features (Questions, Transcripts, Chat, Knowledge, etc.) originally used bare fallback paths like `/questions`, `/transcripts`, `/chat`, etc. None of these top-level routes exist in the app -- all project-scoped routes live under `(dashboard)/projects/[projectId]/...`. Clicking any of these sidebar links when no project was selected would navigate to a non-existent route and produce a 404 page. The test fixture at `tests/src/components/layout/sidebar.tsx` still contains the old buggy fallback paths.

## Location

- `tests/src/components/layout/sidebar.tsx:39` — Old fallback path `/questions` (and 14 similar fallback paths on lines 48, 56, 64, 72, 80, 88, 96, 104, 112, 119, 129, 137, 145, 151) still present in test fixture
- `src/components/layout/sidebar.tsx:38-161` — **Already fixed** in source: fallback paths changed to `#` with `disabled: true`, and disabled items render as `<span>` instead of `<Link>`

## Root Cause

The original `buildNavItems` function used bare top-level routes (e.g., `/questions`, `/transcripts`, `/dashboard`) as fallback `href` values when `activeProjectId` was undefined. These routes do not exist in the Next.js App Router file structure -- all feature pages are nested under `src/app/(dashboard)/projects/[projectId]/`. There was also no disabled state to prevent navigation, so every nav item was always rendered as a clickable `<Link>`.

## Fix

The source file was already fixed in commit `837451d` (and refined in `1fe44a4`). The fix:

1. Changed all fallback `href` values from non-existent routes (e.g., `/questions`) to `"#"`
2. Added `disabled: !activeProjectId` to every project-scoped nav item
3. Added a conditional render path: when `item.disabled` is true, renders a `<span>` with `cursor-not-allowed` styling and a "Select a project first" tooltip instead of a `<Link>`

**Remaining cleanup:** Update the test fixture at `tests/src/components/layout/sidebar.tsx` to match the current source. It still contains the old buggy code with bare fallback paths and no `disabled` property, which means any tests relying on this fixture would not reflect the actual component behavior.
