# BUG-047: Assigned To dropdown shows no members or displays raw user IDs

- **Severity:** HIGH
- **Phase:** 2 (Discovery and Knowledge Brain)
- **Status:** Fixed

## Description

When asking a question, the "Assigned To" dropdown field doesn't work properly. The dropdown either shows no values at all, or when a blank space is selected, it displays a long string of characters (likely a raw user ID or cuid) instead of a human-readable name. The user cannot meaningfully assign questions to team members.

## Location

- `src/components/questions/question-form.tsx:235-254` ‚Äî Assigned To select field
- Parent component that passes `teamMembers` prop to `QuestionForm`

## Root Cause

The QuestionForm component expects a `teamMembers` prop of type `Array<{ id: string; displayName: string; email: string }>`. Possible causes:

1. **Empty teamMembers prop:** The parent component may not be passing the `teamMembers` array, or the query to fetch team members may return an empty array. The field is conditionally rendered (`teamMembers.length > 0`), so if any members are passed, the dropdown shows ‚Äî but they may have empty `displayName` values.

2. **Null displayName:** Related to BUG-005 (previously fixed), where project member records had blank `displayName` and `email` fields. If existing records weren't backfilled, the SelectItem renders `{member.displayName || member.email}` which could be empty or show the raw `member.id` (cuid).

3. **SelectItem value leak:** The `SelectItem` uses `value={member.id}` (a cuid string). If displayName and email are both empty, the SelectValue may fall back to displaying the cuid value.

## Fix

1. Verify the parent component passes `teamMembers` correctly by checking the page/form that renders `QuestionForm`
2. Ensure the team members query returns `displayName` and `email` fields populated
3. Backfill any ProjectMember records with blank `displayName`/`email` from Clerk user data
4. Add a fallback display: `{member.displayName || member.email || "Unknown Member"}` to prevent raw IDs from showing
5. Debug why the dropdown appears empty ‚Äî trace the data flow from the server component through to the form prop
