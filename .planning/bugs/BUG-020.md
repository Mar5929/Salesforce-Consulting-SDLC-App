# BUG-020: Org connection isSyncing always false

- **Severity:** MEDIUM
- **Phase:** 4 (Salesforce Org Connectivity)
- **Status:** Open

## Description

The `OrgConnectionCard` component accepts an `isSyncing` prop to show the "Syncing" status badge with a pulse animation. However, the parent page (`OrgSettingsPage`) hardcodes `isSyncing={false}` and never queries the actual sync run status. This means users never see the syncing indicator, even when a metadata sync is actively running via Inngest. The card always shows "Connected" instead of "Syncing" during active sync runs.

## Location

- `src/app/(dashboard)/projects/[projectId]/settings/org/page.tsx:78` — `isSyncing={false}` is hardcoded

## Root Cause

The server component that renders `OrgConnectionCard` never checks whether an active `OrgSyncRun` exists with status `RUNNING` or `PENDING`. Line 78 passes the literal value `false` for the `isSyncing` prop. No query against the `OrgSyncRun` table is performed to determine the actual sync state.

## Fix

In `src/app/(dashboard)/projects/[projectId]/settings/org/page.tsx`:

1. Add a Prisma query alongside the existing `Promise.all` to check for an active sync run:
   ```ts
   const activeSyncRun = await prisma.orgSyncRun.findFirst({
     where: { projectId, status: { in: ["PENDING", "RUNNING"] } },
   })
   ```
2. Replace `isSyncing={false}` with `isSyncing={!!activeSyncRun}` on the `OrgConnectionCard` props.
3. Consider adding SWR polling or `router.refresh()` on an interval so the page updates when the sync completes.

## Related Files

- `src/actions/org-connection.ts` — `requestSync` dispatches the Inngest event but does not return sync run state
- `prisma/schema.prisma:945` — `OrgSyncRun` model with `SyncRunStatus` enum (PENDING, RUNNING, COMPLETED, FAILED)
