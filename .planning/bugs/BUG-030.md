# BUG-030: Document detail nests anchor inside button

- **Severity:** LOW
- **Phase:** 5 (Documents, QA, Admin)
- **Status:** Fixed

## Description

On the document detail page, the download button wraps an `<a>` tag inside a `<Button>` component (which renders as `<button>`). This produces invalid HTML: `<button><a href="...">...</a></button>`. Browsers handle this inconsistently -- some ignore the anchor click, others fire both button and anchor events. Screen readers also misinterpret the nested interactive elements, creating an accessibility violation (WCAG interactive content nesting).

## Location

- `src/app/(dashboard)/projects/[projectId]/documents/[documentId]/page.tsx:103-116` -- `<Button>` wrapping an `<a>` tag

## Root Cause

The download button was implemented by placing an `<a>` element inside a `<Button>` component. The `<Button>` has `onClick={undefined}` (no-op) and relies on the inner anchor for navigation, but HTML spec does not allow interactive content (`<a>`) inside interactive content (`<button>`).

## Fix

Replace the `<Button>` + nested `<a>` with one of these approaches:

1. **Use `asChild` pattern (preferred):** Use the Button component's `asChild` prop to render the button styling on the anchor element directly:
   ```tsx
   <Button asChild className="bg-[#2563EB] hover:bg-[#1d4ed8] text-white">
     <a href={downloadUrl} target="_blank" rel="noopener noreferrer">
       <Download className="h-4 w-4" />
       Download {formatLabel}
     </a>
   </Button>
   ```

2. **Use Link component:** Replace with a Next.js `Link` styled as a button, or a plain `<a>` with button classes applied via `cn()`.

## Related Files

- `src/components/documents/document-preview.tsx` -- The DOCX/PPTX preview card uses `onClick={() => window.open(...)}` instead, which is the correct pattern for a button that opens a URL
- `src/components/ui/button.tsx` -- Button component that supports `asChild` via Radix Slot
