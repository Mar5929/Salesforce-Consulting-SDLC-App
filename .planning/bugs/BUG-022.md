# BUG-022: Analyze Org button visible to all roles, errors for non-SA

- **Severity:** MEDIUM
- **Phase:** 4 (Salesforce Org Connectivity)
- **Status:** Open

## Description

The "Analyze Org" button on the Org Analysis page is rendered for all project members regardless of role. However, the `triggerIngestion` server action enforces SA-only access (line 25: `role: "SOLUTION_ARCHITECT"`). When a non-SA member (BA, QA, Developer, PM) clicks the button, they receive a cryptic server error toast ("Only Solution Architects can trigger org ingestion") instead of the button being hidden or disabled. The same issue affects the "Re-analyze" button shown after an initial analysis has run.

## Location

- `src/app/(dashboard)/projects/[projectId]/org/analysis/analysis-client.tsx:137-148` — "Analyze Org" button rendered without role check
- `src/app/(dashboard)/projects/[projectId]/org/analysis/analysis-client.tsx:163-175` — "Re-analyze" button also rendered without role check
- `src/app/(dashboard)/projects/[projectId]/org/analysis/page.tsx:22` — `getCurrentMember` is called but the member's role is not passed to the client

## Root Cause

The `OrgAnalysisPage` server component calls `getCurrentMember(projectId)` on line 22 but discards the result -- it only uses it for the implicit membership check. The member's role is never passed down to `AnalysisClient`, so the client component has no way to conditionally render the button. The `AnalysisClientProps` interface does not include a role or permission flag.

## Fix

1. In `src/app/(dashboard)/projects/[projectId]/org/analysis/page.tsx`, capture the member role from `getCurrentMember`:
   ```ts
   const currentMember = await getCurrentMember(projectId)
   const canTriggerAnalysis = currentMember.role === "SOLUTION_ARCHITECT"
   ```

2. Pass `canTriggerAnalysis` as a prop to `AnalysisClient` (through `AnalysisData`).

3. In `src/app/(dashboard)/projects/[projectId]/org/analysis/analysis-client.tsx`:
   - Add `canTriggerAnalysis: boolean` to `AnalysisClientProps`
   - Wrap both "Analyze Org" and "Re-analyze" buttons with `{canTriggerAnalysis && ...}` conditionals

4. Alternatively, if PMs should also be able to trigger analysis (per the `verifyAnalysisRole` pattern used elsewhere), update `triggerIngestion` to accept PM role as well, and gate the button on `["SOLUTION_ARCHITECT", "PM"].includes(role)`.

## Related Files

- `src/actions/org-analysis.ts:14-30` — `triggerIngestion` enforces SA-only; decides whether PM should also have access
- `src/lib/auth.ts` — `getCurrentMember` returns the member record with role
- `src/app/(dashboard)/projects/[projectId]/settings/org/page.tsx:38` — Example of the correct pattern: `canManage` derived from role and passed to child component
