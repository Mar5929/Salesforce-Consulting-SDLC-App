# BUG-037: Chat window has no scroll — expands entire page height

- **Severity:** HIGH
- **Phase:** 2 (Discovery and Knowledge Brain)
- **Status:** Fixed

## Description

When a chat conversation is open, the chat message area expands the entire length and height of the page instead of being constrained to a scrollable container. There is no ability to scroll within the chat window itself — the whole page scrolls instead. This makes the chat unusable for longer conversations.

## Location

- `src/components/chat/message-list.tsx:71-85` — Message list with ScrollArea and auto-scroll logic
- `src/components/chat/chat-interface.tsx` — Parent chat component layout
- `src/components/chat/chat-layout.tsx:14-28` — Outer layout container

## Root Cause

The flex layout containing the message list likely doesn't have proper height constraints. The message list's ScrollArea needs a fixed or constrained height (e.g., `h-full overflow-hidden` on the parent with `flex-1 overflow-y-auto` on the message container). Without these constraints, the content grows unbounded, pushing the page height instead of scrolling within the container.

Additionally, the ScrollArea ref forwarding (`scrollRef`) may not be working correctly, which would break the auto-scroll-to-bottom behavior even if the container were properly constrained.

## Fix

1. Ensure the chat layout has `h-full` or `h-screen` with `overflow-hidden` on the outer container
2. Set `flex-1 min-h-0 overflow-hidden` on the message list's parent to allow flex shrinking
3. Verify ScrollArea properly constrains its content and enables internal scrolling
4. Test with a long conversation to confirm scroll behavior works
