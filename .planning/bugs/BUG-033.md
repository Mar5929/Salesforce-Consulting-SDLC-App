# BUG-033: Chat "Response interrupted" — UIMessage content extraction fails

- **Severity:** HIGH
- **Phase:** 2
- **Status:** Fixed

## Description

Sending a message in Project Chat fails with "Response interrupted. Click to retry." The AI never responds. The browser console shows a 500 error from `/api/chat` with: `Argument 'content' is missing` on `prisma.chatMessage.create()`.

## Location

- `src/app/api/chat/route.ts:90-104` — Reads `lastUserMessage.content` which is `undefined` in AI SDK v6 UIMessage format

## Root Cause

AI SDK v6's `useChat` sends messages in `UIMessage` format with a `parts` array instead of the legacy `{ role, content }` format. The chat API route assumed `lastUserMessage.content` would be a string, but it was `undefined`. This caused the Prisma `chatMessage.create()` call to fail because `content` is a required field.

## Fix

Extract text content from the `parts` array when `content` is not a string. Also added `convertToModelMessages()` to convert UIMessages to ModelMessages for the `streamText` call.

Fixed in commit `7080b1e`.
