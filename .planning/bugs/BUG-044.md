# BUG-044: Bulk status change operations do nothing

- **Severity:** HIGH
- **Phase:** 3 (Story Management and Sprint Intelligence)
- **Status:** Fixed

## Description

In the Work tab, selecting multiple items (features or stories within an epic) and using the bulk operation to change their status to "Done" does nothing. The status doesn't change, no error is shown, and no feedback is provided. The bulk action appears completely non-functional.

## Location

- `src/app/(dashboard)/projects/[projectId]/work/[epicId]/epic-detail-client.tsx` — Epic detail page where bulk operations on features would occur
- `src/app/(dashboard)/projects/[projectId]/work/[epicId]/[featureId]/feature-detail-client.tsx` — Feature detail page where bulk operations on stories would occur

## Root Cause

Needs investigation. Possible causes:
1. The bulk action handler may not be wired up to a server action
2. The server action may exist but fail silently without error handling
3. The checkbox selection state may not be properly tracked or passed to the bulk action
4. The status update may succeed in the database but the UI doesn't refresh to reflect the change

## Fix

1. Verify that bulk status change buttons are connected to server actions
2. Ensure server actions for bulk status updates exist and handle the correct payload
3. Add error handling with toast notifications for success/failure
4. Force a UI refresh (router.refresh() or revalidatePath) after successful bulk update
5. Add loading state to the bulk action button during execution

## Related Files

- `src/actions/stories.ts` — Story actions (check for bulk update action)
- `src/actions/features.ts` or similar — Feature actions
- `src/components/work/` — Work components with table views and bulk selection
