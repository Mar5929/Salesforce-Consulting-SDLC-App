# BUG-005: Project creator member record has blank displayName/email

- **Severity:** HIGH
- **Phase:** 1 (Foundation)
- **Status:** Open

## Description

When a user creates a new project, the `ProjectMember` record for the creator is saved with `displayName: ""` and `email: ""` (hardcoded empty strings). This means the creator's name and email are blank throughout the app anywhere member info is displayed -- question ownership, team lists, activity logs, etc. The invited team members are correctly populated by looking up Clerk user data, but this lookup is skipped for the creator.

## Location

- `src/actions/projects.ts:62-70` -- `ProjectMember.create` for the creator uses hardcoded `displayName: ""` and `email: ""`

## Root Cause

The `createProject` action has `ctx.userId` (the Clerk user ID) available from the safe-action middleware, but it never calls the Clerk API to fetch the user's profile (name, email) before creating the member record. The code for invited team members (lines 76-99) correctly calls `clerk.users.getUserList()` and populates `displayName` and `email` from the Clerk response, but the creator's record skips this step entirely and writes empty strings.

## Fix

In `src/actions/projects.ts`, before creating the creator's `ProjectMember` record, fetch the creator's Clerk profile and populate the fields:

1. Move the Clerk client import (`clerkClient`) above the creator member creation (currently it is imported inside the `if (parsedInput.teamMembers)` block at line 76-77).

2. Fetch the creator's user data:
   ```ts
   const { clerkClient } = await import("@clerk/nextjs/server")
   const clerk = await clerkClient()
   const creatorUser = await clerk.users.getUser(ctx.userId)
   ```

3. Update the creator member creation (line 62-70) to use the fetched data:
   ```ts
   await prisma.projectMember.create({
     data: {
       projectId: project.id,
       clerkUserId: ctx.userId,
       displayName:
         `${creatorUser.firstName ?? ""} ${creatorUser.lastName ?? ""}`.trim() ||
         creatorUser.emailAddresses[0]?.emailAddress || "",
       email: creatorUser.emailAddresses[0]?.emailAddress || "",
       role: "SOLUTION_ARCHITECT",
       status: "ACTIVE",
     },
   })
   ```

4. Reuse the same `clerk` instance for the invited team members block below to avoid a redundant import.

- `src/lib/auth.ts` -- `getCurrentMember()` returns the member record including `displayName` and `email`
- `prisma/schema.prisma` -- ProjectMember model definition with `displayName` and `email` fields
