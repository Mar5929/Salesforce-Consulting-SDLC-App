# BUG-012: Jira field customfield_story_points is invalid

- **Severity:** HIGH
- **Phase:** 5 (Documents, QA, Admin)
- **Status:** Open

## Description

When syncing stories with story points to Jira, the `mapStoryToJiraFields` function uses the hardcoded field name `customfield_story_points`. Jira custom fields use numeric IDs like `customfield_10016`, not semantic names. This causes the Jira API to reject issue creation/update requests with a field validation error whenever a story has story points set.

## Location

- `src/lib/jira/field-mapping.ts:47` -- `customfield_story_points` is not a valid Jira field identifier

## Root Cause

Jira custom fields are identified by `customfield_NNNNN` (numeric ID), not by human-readable names. The story points custom field ID varies per Jira instance -- there is no universal field name. The code hardcodes a made-up identifier `customfield_story_points` which no Jira instance will recognize.

## Fix

Two options, from simplest to most robust:

1. **Remove story points from initial sync (quick fix):** Remove the `customfield_story_points` spread in `mapStoryToJiraFields` and add a TODO to implement proper field discovery. This unblocks story sync immediately.

2. **Add configurable story points field ID (proper fix):** Add a `storyPointsFieldId` column to the `JiraConfig` model (e.g., `customfield_10016`). In the Jira config form, either let the user enter the field ID manually or add a field discovery step that calls `GET /rest/api/3/field` to list available fields and lets the user pick the story points field. Then use that configured ID in `mapStoryToJiraFields`.

## Related Files

- `src/lib/jira/sync.ts` -- Calls `mapStoryToJiraFields` and passes result to Jira API
- `src/lib/jira/client.ts` -- Creates the Jira API client
- `src/components/jira/jira-config-form.tsx` -- Would need a new field for story points field ID
