# BUG-036: Archive button on conversations may not work

- **Severity:** MEDIUM
- **Phase:** 2 (Discovery and Knowledge Brain)
- **Status:** Open

## Description

The archive button on chat conversations may not function correctly. User was uncertain whether it works at all — clicking it doesn't produce a clear result or confirmation.

## Location

- `src/components/chat/conversation-sidebar.tsx:110-141` — Archive/unarchive functionality
- `src/actions/conversations.ts:227-280` — `archiveConversation` and `unarchiveConversation` server actions

## Root Cause

Needs investigation. Possible issues:
1. The archive action may succeed silently without visual feedback (no toast notification or UI update)
2. The conversation may be archived in the database but the sidebar list doesn't refresh to remove it
3. The show/hide archived toggle (lines 252-260) may not be obvious, so the user can't see archived conversations to verify the action worked

## Fix

1. Verify the archive server action is wired up correctly and actually called on button click
2. Add a toast notification on successful archive (e.g., "Conversation archived" with undo option)
3. Ensure the sidebar list refreshes after archiving to remove the conversation from the active list
4. Make the "Show archived" toggle more visible so users can verify archiving worked
