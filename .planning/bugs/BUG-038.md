# BUG-038: New Session type buttons (Story Generation, etc.) do nothing on click

- **Severity:** HIGH
- **Phase:** 2 (Discovery and Knowledge Brain)
- **Status:** Fixed

## Description

Clicking "New Session" and selecting any session type (Story Generation, Briefing, Question Impact, Story Enrichment) does nothing. No new session is created, no UI changes, no navigation occurs.

## Location

- `src/components/chat/new-session-dropdown.tsx:53-69` — Session type click handlers
- `src/components/chat/chat-interface.tsx` — Does not process `?newSession` search param

## Root Cause

The `new-session-dropdown.tsx` component handles session creation differently by type:
- **Briefing sessions:** Create immediately via `createConversation()` action and navigate to the new conversation
- **Story Generation, Question Impact, Enrichment:** Navigate to the chat page with a `?newSession={type}` URL search parameter, expecting the chat interface to show an entity selection UI (e.g., pick an epic for story generation)

The problem is that `chat-interface.tsx` never reads or processes the `?newSession` search parameter. There is no entity selection UI implemented. So clicking these session types adds a query param to the URL but nothing happens visually.

## Fix

1. In `chat-interface.tsx`, read the `newSession` search parameter
2. When present, display an entity selection UI:
   - Story Generation: show epic/feature picker
   - Question Impact: show question picker
   - Story Enrichment: show story picker
3. After entity selection, call the appropriate `initiateXxxSession()` action to create the conversation
4. Navigate to the newly created conversation

Alternatively, move the entity selection into the dropdown itself as a sub-menu or modal.
