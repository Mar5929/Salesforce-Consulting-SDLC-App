# BUG-034: Projects tab shows no content — only displays project name

- **Severity:** MEDIUM
- **Phase:** 1 (Foundation and Data Layer)
- **Status:** Open

## Description

Clicking the "Projects" tab in the left sidebar shows a mostly empty page. The only visible content is the project name "Salesforce SF Consulting AI Framework" with no additional details, metrics, or actionable content. The page feels like a dead end — there's no useful project overview information.

## Location

- `src/app/(dashboard)/projects/[projectId]/page.tsx` — Project detail page that renders after clicking a project card
- `src/app/(dashboard)/page.tsx` — Root dashboard page that shows project cards

## Root Cause

The Projects sidebar tab routes to `/` which shows project cards. Clicking a card navigates to `/projects/{projectId}/page.tsx` which only renders basic info: project name, client name, engagement type badge, current phase badge, start/end dates, and quick links to Team and Settings. There is no dashboard summary, no metrics, no recent activity — just sparse metadata fields.

## Fix

Enhance the project detail page (`/projects/[projectId]/page.tsx`) with a proper project overview dashboard:
1. Add key metrics cards (questions count, stories count, knowledge articles, team members)
2. Add recent activity feed
3. Add project health/progress indicators
4. Add quick-action buttons for common tasks

Alternatively, if the sidebar "Projects" tab should show a project list rather than navigating into a single project, adjust the sidebar routing.
