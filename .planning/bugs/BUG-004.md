# BUG-004: Sidebar badge counts ANSWERED instead of needsReview

- **Severity:** HIGH
- **Phase:** 1 (Foundation)
- **Status:** Open

## Description

The sidebar badge on the "Questions" nav item shows the count of questions with `status: "ANSWERED"` instead of questions with `needsReview: true`. The Questions page itself correctly counts `needsReview: true` (line 61), but the layout that feeds the sidebar badge uses the wrong filter. This means the badge may show a misleading count -- questions that are answered but do not need review inflate the number, while non-ANSWERED questions flagged for review (e.g., by the AI agent harness via `flag-conflict` tool) are missed.

## Location

- `src/app/(dashboard)/layout.tsx:37-38` -- Queries `prisma.question.count({ where: { projectId, status: "ANSWERED" } })` instead of filtering on `needsReview: true`

## Root Cause

When the sidebar badge query was written, the developer used `status: "ANSWERED"` as a proxy for "needs review." However, the schema has a dedicated `needsReview` boolean field on the Question model (at `prisma/schema.prisma:620`). The `status` field and `needsReview` field are independent -- a question can be ANSWERED without needing review, and a question can need review without being in ANSWERED status (e.g., AI-flagged conflicts set `needsReview: true` on questions that may still be in other statuses).

## Fix

In `src/app/(dashboard)/layout.tsx`, change the question count query (line 37-38) from:

```ts
prisma.question.count({
  where: { projectId, status: "ANSWERED" },
})
```

to:

```ts
prisma.question.count({
  where: { projectId, needsReview: true },
})
```

This matches the query used on the Questions page itself at `src/app/(dashboard)/projects/[projectId]/questions/page.tsx:61`.

Also update the comment on line 36 from "Count questions answered..." to "Count questions flagged for human review."
