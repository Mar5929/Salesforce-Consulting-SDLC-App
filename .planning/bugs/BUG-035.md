# BUG-035: Chat tab shows stale pre-feature conversation, new chats missing from sidebar

- **Severity:** HIGH
- **Phase:** 2 (Discovery and Knowledge Brain)
- **Status:** Open

## Description

Clicking the "Chat" tab in the left sidebar shows an odd UI state. The conversations sidebar panel appears, but the main area opens a chat that doesn't appear in the conversations list. This appears to be a legacy conversation created before the conversations/session management feature was built. New conversations created through the chat don't show up in the sidebar conversations list either.

## Location

- `src/components/chat/conversation-sidebar.tsx` — Conversations list panel
- `src/components/chat/chat-layout.tsx:14-28` — Split layout (320px sidebar + chat area)
- `src/app/(dashboard)/projects/[projectId]/chat/page.tsx` — Chat page entry point

## Root Cause

Likely causes:
1. Legacy conversations created before the session management feature don't have proper `conversationType` values, so they don't match the sidebar's filtering logic
2. The sidebar filters conversations by type (lines 62-89) and may exclude conversations that don't match expected session types
3. The chat page may auto-load the most recent conversation by ID regardless of whether it appears in the filtered sidebar list

## Fix

1. Ensure the conversation sidebar query includes all conversation types, or has a "General" fallback category
2. Handle legacy conversations that predate the session type system — either migrate them to a default type or show them in an "Other" section
3. Verify that newly created conversations appear immediately in the sidebar list (may need router refresh or optimistic update)
