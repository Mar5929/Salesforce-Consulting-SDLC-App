# BUG-017: Knowledge article summary never updated during refresh

- **Severity:** MEDIUM
- **Phase:** 2 (Discovery and Knowledge)
- **Status:** Fixed

## Description

When a stale knowledge article is refreshed by the `article-refresh` Inngest function, the article's `content` field is updated with the new AI-synthesized text, but the `summary` field is never updated. The `summary` field (a required `String` in the Prisma schema) retains whatever value it had from the original article creation. Over time, as articles are refreshed with new source material, the summary becomes increasingly stale and misleading relative to the actual content. The summary is displayed prominently in the article list view (`getArticles` action) and is the primary preview users see.

## Location

- `src/lib/inngest/functions/article-refresh.ts:84-97` — The `update-article` step updates `content`, `version`, `isStale`, `staleReason`, `staleSince`, `lastRefreshedAt`, and `embeddingStatus`, but does not update `summary`
- `src/lib/agent-harness/tasks/article-synthesis.ts:144-147` — The task output format only asks for updated article content as markdown; does not request a separate summary

## Root Cause

The article refresh pipeline has two gaps: (1) the `articleSynthesisTask` prompt (line 130-147) only asks the AI to return updated article content, with no instruction to also produce a summary, and (2) the `update-article` step in the Inngest function (line 85-97) only writes `content: synthesisResult.output` without touching the `summary` field. The `summary` column is required (`String`, not `String?`) in the schema, so it always has a value -- but that value is frozen from initial creation.

## Fix

1. Update the `articleSynthesisTask` system prompt in `src/lib/agent-harness/tasks/article-synthesis.ts` to request a structured output that includes both updated content and a 1-2 sentence summary. For example, ask the AI to return a JSON block with `{ content, summary }` or use a delimiter format.

2. In `src/lib/inngest/functions/article-refresh.ts`, parse the synthesis output to extract both `content` and `summary`, then include `summary` in the Prisma update:
```typescript
data: {
  content: parsedContent,
  summary: parsedSummary,
  version: { increment: 1 },
  // ... rest of fields
}
```

3. Alternatively, add a lightweight post-processing step that generates a summary from the refreshed content using a short AI call (cheaper than changing the synthesis prompt format).

## Related Files

- `src/lib/agent-harness/tasks/article-synthesis.ts` — Task definition with the system prompt and context loader
- `src/actions/knowledge.ts` — `getArticles` action (line 54) selects and returns `summary` to the UI
