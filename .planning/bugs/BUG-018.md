# BUG-018: Backlog page never passes sprints to StoryTable

- **Severity:** MEDIUM
- **Phase:** 3 (Story Management)
- **Status:** Open

## Description

The backlog page's bulk action toolbar is supposed to let users assign selected stories to a sprint. However, the "Assign Sprint" dropdown never appears because the `sprints` prop is never passed to `StoryTable`. The server component (`page.tsx`) does not query sprints at all, the client component (`backlog-client.tsx`) does not accept a `sprints` prop, and the `StoryTable` is rendered without it. Since `StoryTable` defaults `sprints` to `[]`, the condition `sprints.length > 0` on line 382 is always false, and the sprint assignment select is never rendered.

## Location

- `src/app/(dashboard)/projects/[projectId]/backlog/page.tsx:25-67` -- Server component never queries sprints from the database
- `src/app/(dashboard)/projects/[projectId]/backlog/backlog-client.tsx:21-28` -- `BacklogClientProps` interface has no `sprints` field
- `src/app/(dashboard)/projects/[projectId]/backlog/backlog-client.tsx:78-83` -- `StoryTable` rendered without `sprints` prop
- `src/components/work/story-table.tsx:382-395` -- Sprint assignment select guarded by `sprints.length > 0`

## Root Cause

The backlog page server component fetches stories, epics, features, and members, but omits a query for sprints. The `BacklogClientProps` interface does not include a `sprints` field, so even if sprints were fetched, there is no way to pass them through to `StoryTable`. This is a straightforward omission -- the sprint bulk-assign feature was wired in `StoryTable` but the data pipeline from page to table was never connected for the backlog route.

## Fix

1. In `src/app/(dashboard)/projects/[projectId]/backlog/page.tsx`, add a Prisma query to fetch active/planned sprints:
   ```ts
   const sprints = await prisma.sprint.findMany({
     where: { projectId, status: { in: ["PLANNED", "ACTIVE"] } },
     select: { id: true, name: true },
     orderBy: { startDate: "asc" },
   })
   ```

2. Pass `sprints` to `<BacklogClient>`.

3. In `src/app/(dashboard)/projects/[projectId]/backlog/backlog-client.tsx`, add `sprints: Array<{ id: string; name: string }>` to `BacklogClientProps`.

4. Pass `sprints={sprints}` from `BacklogClient` to `<StoryTable>` on line 78.
