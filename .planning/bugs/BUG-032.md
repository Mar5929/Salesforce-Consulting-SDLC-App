# BUG-032: Project switcher dropdown doesn't match single-project workflow

- **Severity:** MEDIUM
- **Phase:** 1 (Foundation)
- **Status:** Open

## Description

The project switcher dropdown always renders as a dropdown menu, even when the user has only one project. For the common single-project workflow, this means the user sees a dropdown trigger with a chevron icon that, when clicked, shows a single item they are already on. This is unnecessary UI friction and visually misleading -- it suggests there are multiple projects to choose from. Additionally, the dropdown shows "Select Project" as placeholder text when no project is active in the URL, but does not auto-navigate to the sole project.

## Location

- `src/components/layout/project-switcher.tsx:19-101` -- `ProjectSwitcher` component always renders `DropdownMenu` regardless of project count
- `src/components/layout/project-switcher.tsx:66-68` -- Shows "Select Project" even when only one project exists

## Root Cause

The `ProjectSwitcher` component was built with a multi-project assumption. It fetches all projects from `/api/projects`, but has no conditional rendering based on the project count. When `projects.length === 1`, it still renders the full dropdown UI with chevron affordance. There is also no logic to auto-select the only available project when the user is not on a project-scoped route.

## Fix

In `src/components/layout/project-switcher.tsx`:

**1. Auto-navigate when single project and no active project:**

After fetching projects, if there is exactly one project and no `activeProjectId` in the URL, auto-navigate to that project:

```ts
useEffect(() => {
  if (projects.length === 1 && !activeProjectId) {
    router.replace(`/projects/${projects[0].id}`)
  }
}, [projects, activeProjectId, router])
```

**2. Render static display for single project:**

When `projects.length === 1`, skip the dropdown and render a simple static display:

```tsx
if (projects.length === 1) {
  return (
    <div className="px-4 py-3">
      <div className="px-2 py-1.5">
        <div className="text-sm font-semibold truncate">{projects[0].name}</div>
        <div className="text-xs text-muted-foreground">{projects[0].clientName}</div>
      </div>
    </div>
  )
}
```

**3. Keep the dropdown for multi-project users (2+ projects):**

The existing dropdown logic remains for users with multiple projects.
