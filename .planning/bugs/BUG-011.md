# BUG-011: Archive redirect goes to /projects which doesn't exist

- **Severity:** HIGH
- **Phase:** 5 (Documents, QA, Admin)
- **Status:** Fixed

## Description

After successfully archiving a project, the `ProjectLifecycleSection` component calls `router.push("/projects")`. However, there is no route at `/projects` -- the project list is rendered by the dashboard root page at `/(dashboard)/page.tsx`, which maps to the URL path `/`. Users see a 404 page after archiving a project instead of being returned to the project list.

## Location

- `src/app/(dashboard)/projects/[projectId]/settings/project-lifecycle-section.tsx:51` -- `router.push("/projects")` navigates to a non-existent route

## Root Cause

The `/(dashboard)/` route group has `page.tsx` at its root (serving `/`), and `projects/[projectId]/` for individual project pages, but there is no `projects/page.tsx` to handle the `/projects` URL. The archive success callback uses `/projects` assuming a projects list page exists at that path, but the project list is actually the dashboard root at `/`.

## Fix

In `src/app/(dashboard)/projects/[projectId]/settings/project-lifecycle-section.tsx`, change line 51 from `router.push("/projects")` to `router.push("/")` to redirect to the dashboard root where the project list lives.

## Related Files

- `src/app/(dashboard)/page.tsx` -- The actual project list page (dashboard root)
- `src/actions/project-archive.ts` -- The server action (returns data but does not control redirect)
