# BUG-039: Work tab UI is clunky and unclear — hard to tell epics from features

- **Severity:** MEDIUM
- **Phase:** 3 (Story Management and Sprint Intelligence)
- **Status:** Fixed

## Description

The Work tab UI is confusing and clunky. It displays rows of items but it's unclear whether they are epics, features, or stories. The hierarchy (Epics > Features > Stories) is not visually communicated. The layout feels like raw data rows rather than a polished work management interface.

## Location

- `src/app/(dashboard)/projects/[projectId]/work/work-page-client.tsx:1-51` — Work page with table/kanban toggle
- `src/components/work/epic-kanban.tsx` — Epic kanban view
- `src/app/(dashboard)/projects/[projectId]/work/[epicId]/epic-detail-client.tsx` — Epic detail with features

## Root Cause

The Work page renders epics in either a table or kanban view without strong visual hierarchy indicators. The table rows don't clearly label items as "Epics" vs "Features." When navigating into an epic, the features inside also lack clear hierarchy breadcrumbs or labels. The overall UX doesn't guide the user through the Epic > Feature > Story drill-down pattern.

## Fix

1. Add breadcrumbs to show current hierarchy level (Work > Epic Name > Feature Name)
2. Add visual indicators (icons, labels, or color coding) to distinguish epics, features, and stories
3. Add summary counts on each level (e.g., "3 Features, 12 Stories" on an epic card)
4. Improve table column headers and row styling for better scannability
5. Consider adding a sidebar tree view for navigation between hierarchy levels
