# BUG-021: Brownfield pipeline stepper never shows running state

- **Severity:** MEDIUM
- **Phase:** 4 (Salesforce Org Connectivity)
- **Status:** Fixed

## Description

The `PipelineStepper` component correctly supports `status: "running"` with a pulse animation and "In progress..." label, but the parent `AnalysisData` server component never passes `"running"` as the pipeline status. It uses a simple binary check: if domain groupings exist, status is `"completed"` with phase 4; otherwise, status is `"idle"` with phase 0. After the user clicks "Analyze Org" and the Inngest ingestion pipeline begins processing, the stepper remains stuck showing either the idle state or the completed state, never showing intermediate progress.

## Location

- `src/app/(dashboard)/projects/[projectId]/org/analysis/page.tsx:81-82` — `pipelinePhase` and `pipelineStatus` are derived with only two possible values (`0/idle` or `4/completed`)

## Root Cause

The `AnalysisData` server component does not track the ingestion pipeline's actual run state. Lines 81-82 use a simplistic binary:
```ts
const pipelinePhase = hasRun ? 4 : 0
const pipelineStatus = hasRun ? "completed" : "idle"
```

There is no database field or query that tracks which pipeline step the Inngest function (`org-ingestion`) is currently executing. The Inngest step function runs steps named `parse-org-components`, `classify-components`, `synthesize-business-processes`, and `finalize-ingestion`, but their progress is not persisted to the database.

## Fix

1. **Add pipeline tracking to the database**: Either add `ingestionStatus` and `ingestionPhase` fields to the `Project` model, or create a dedicated `OrgIngestionRun` model (similar to `OrgSyncRun`) with status and current phase fields.

2. **Update the Inngest function** (`src/lib/inngest/functions/org-ingestion.ts`): At the start of each step, update the database record with the current phase (1=Parse, 2=Classify, 3=Synthesize, 4=Complete) and status (`running`/`completed`/`failed`).

3. **Update the server component** (`src/app/(dashboard)/projects/[projectId]/org/analysis/page.tsx`): Query the ingestion run status from the database instead of using the binary `hasRun` check. Pass the real `pipelinePhase` and `pipelineStatus` to `AnalysisClient`.

4. **Add client-side polling**: After the user clicks "Analyze Org", poll with SWR or `router.refresh()` on an interval to pick up phase transitions as the Inngest function progresses.

## Related Files

- `src/components/org/pipeline-stepper.tsx` — Stepper component renders correctly for all states (no changes needed)
- `src/lib/inngest/functions/org-ingestion.ts` — Inngest step function with 4 phases; needs to persist phase progress
- `src/app/(dashboard)/projects/[projectId]/org/analysis/analysis-client.tsx` — Client component that passes stepper props; receives data from server component
