# BUG-028: PM Dashboard accessible by URL, shows infinite loading

- **Severity:** LOW
- **Phase:** 5 (Documents, QA, Admin)
- **Status:** Open

## Description

Any project member can navigate directly to `/projects/{id}/pm-dashboard` regardless of their role. The page renders with a heading and skeleton loader. The `getPmDashboardData` server action throws "PM Dashboard requires PM or SA role" for non-PM/SA users, but the client component (`PmDashboardClient`) does not handle this error. SWR receives `null` from the failed action and the component shows the "Dashboard is building" empty state indefinitely, with SWR re-polling every 30 seconds and silently failing each time.

## Location

- `src/app/(dashboard)/projects/[projectId]/pm-dashboard/page.tsx:17-32` -- No server-side role check; renders for any authenticated member
- `src/app/(dashboard)/projects/[projectId]/pm-dashboard/pm-dashboard-client.tsx:30-36` -- `fetchDashboardData` swallows errors by returning `null`
- `src/actions/pm-dashboard.ts:44-45` -- Role check throws but the error is caught silently client-side

## Root Cause

The PM Dashboard page component (`page.tsx`) does not verify the user's role before rendering. It delegates role checking to the server action, but the client-side `fetchDashboardData` wrapper catches the thrown error and returns `null`, which the component interprets as "no data yet" rather than "access denied." There is no distinction between "data not yet generated" and "unauthorized."

## Fix

1. **Add server-side role gate in `page.tsx`:** Import `getCurrentMember` from `@/lib/auth`, check that `member.role` is `PM` or `SOLUTION_ARCHITECT`, and call `notFound()` or redirect for other roles. This prevents the page from rendering at all for unauthorized users.

2. **Improve error handling in the client:** In `fetchDashboardData`, distinguish between authorization errors and missing data. If the action throws a role error, surface an "Access denied" message instead of the "Dashboard is building" empty state.

3. **Hide the sidebar link:** In the sidebar navigation, conditionally show the PM Dashboard link only for PM/SA roles (it may already have a `roles` filter -- verify and ensure it is applied).

## Related Files

- `src/components/layout/sidebar.tsx` -- Sidebar nav items, may need role filtering for PM Dashboard link
- `src/actions/pm-dashboard.ts` -- Server action with role check that throws on unauthorized access
