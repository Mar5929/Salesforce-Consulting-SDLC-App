# BUG-043: Feature details are not editable — no edit UI

- **Severity:** MEDIUM
- **Phase:** 3 (Story Management and Sprint Intelligence)
- **Status:** Open

## Description

When viewing a feature's detail page, there is no way to edit any of the feature's fields. No edit button, no inline editing, no edit form. The page is entirely read-only with no mechanism to update the feature's name, description, status, or any other property.

## Location

- `src/app/(dashboard)/projects/[projectId]/work/[epicId]/[featureId]/feature-detail-client.tsx:1-144` — Feature detail page (read-only)

## Root Cause

The feature detail page was built as a read-only display. No edit form component exists for features. While stories have some editing capabilities, the feature level lacks:
1. An edit button/mode toggle
2. A feature edit form component
3. A server action for updating feature fields

## Fix

1. Create a `FeatureEditForm` component (or use inline editing) for the feature detail page
2. Create an `updateFeature` server action in the features actions file
3. Add an "Edit" button to the feature detail page header that toggles between view and edit modes
4. Support editing: name, description, status, priority, acceptance criteria, and any other editable fields
5. Follow the same edit pattern used elsewhere in the app for consistency
