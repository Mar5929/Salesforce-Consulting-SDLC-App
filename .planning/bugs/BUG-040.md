# BUG-040: Generate Stories button throws Prisma conversation.create error

- **Severity:** HIGH
- **Phase:** 3 (Story Management and Sprint Intelligence)
- **Status:** Open

## Description

Clicking the "Generate Stories" button on the Work tab inside an epic throws an "invalid Prisma conversation.create invocation" error. The story generation session cannot be initiated.

## Location

- `src/app/(dashboard)/projects/[projectId]/work/[epicId]/epic-detail-client.tsx:69-71` — `handleGenerateStories()` button handler
- `src/actions/conversations.ts:309-332` — `initiateStorySession` server action with Prisma create call

## Root Cause

The `initiateStorySession` action calls `prisma.conversation.create()` at lines 321-329:
```typescript
const conversation = await prisma.conversation.create({
  data: {
    projectId,
    conversationType: "STORY_SESSION",
    title: `AI Story Generation - ${epicLabel}`,
    createdById: member.id,
    metadata: { epicId, ...(featureId && { featureId }) },
  },
})
```

Possible causes:
1. The Prisma client may be out of sync with the schema (needs `prisma generate`)
2. The `metadata` field may not accept the JSON shape being passed
3. The `createdById` may reference a ProjectMember that doesn't exist or has been deleted
4. A required field may be missing from the create payload

## Fix

1. Run `npx prisma generate` to ensure the Prisma client matches the current schema
2. Verify the Conversation model schema matches the fields in the create call
3. Check that the `metadata` field is defined as `Json?` in the schema and accepts arbitrary JSON
4. Add error handling with a user-friendly toast message instead of an unhandled action error
