# BUG-013: SF_TOKEN_ENCRYPTION_KEY undocumented -- crypto crash

- **Severity:** HIGH
- **Phase:** 5 (Documents, QA, Admin)
- **Status:** Fixed

## Description

The `SF_TOKEN_ENCRYPTION_KEY` environment variable is required by `src/env.ts` (validated as `z.string().min(32)`) and used in `src/lib/encryption.ts` to derive per-project encryption keys via HKDF. If the variable is missing or too short, the app crashes at startup with an opaque Zod validation error from `@t3-oss/env-nextjs`. While `.env.example` does document this variable, it provides only a placeholder hint (`32-char-minimum-encryption-key-here`) with no generation instructions. The `encryption.ts` module uses `process.env.SF_TOKEN_ENCRYPTION_KEY!` (non-null assertion) without any fallback, so if env validation is bypassed, Node's crypto module throws an unhandled error.

## Location

- `src/lib/encryption.ts:7` -- `process.env.SF_TOKEN_ENCRYPTION_KEY!` non-null assertion with no graceful error
- `src/env.ts:8` -- `SF_TOKEN_ENCRYPTION_KEY: z.string().min(32)` required but no generation docs

## Root Cause

The encryption module reads the env var directly via `process.env` with a non-null assertion instead of importing from the validated `env` object. This means: (1) if the env var is missing, the HKDF call receives `undefined` as the master key, causing a cryptic `TypeError` from Node's crypto module rather than a clear message; (2) there is no documentation explaining what this key should be (a random 32+ byte hex/base64 string) or how to generate one (e.g., `openssl rand -hex 32`).

## Fix

1. In `src/lib/encryption.ts`, import `{ env }` from `@/env` and use `env.SF_TOKEN_ENCRYPTION_KEY` instead of `process.env.SF_TOKEN_ENCRYPTION_KEY!` so the validated value is used consistently.

2. In `.env.example`, add a comment above `SF_TOKEN_ENCRYPTION_KEY` explaining how to generate a valid key:
   ```
   # Generate with: openssl rand -hex 32
   SF_TOKEN_ENCRYPTION_KEY=
   ```

3. Optionally, add a startup guard in `encryption.ts` that throws a descriptive error if the key is missing: `throw new Error("SF_TOKEN_ENCRYPTION_KEY is required for token encryption. Generate with: openssl rand -hex 32")`.

## Related Files

- `src/env.ts` -- Central env validation where the var is declared required
- `.env.example` -- Template for environment setup
- `src/lib/salesforce/oauth.ts` -- Calls encrypt/decrypt for SF tokens
- `src/lib/jira/client.ts` -- Calls decrypt for Jira API tokens
