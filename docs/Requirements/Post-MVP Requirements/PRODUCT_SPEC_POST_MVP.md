# Product Spec: Post-MVP (Phase 2)

**Purpose.** Parking lot for capabilities and behaviors that were considered during `PRODUCT_SPEC.md` review, judged valuable, and intentionally deferred out of MVP scope. Items here do not gate MVP delivery.

**Relationship to `PRODUCT_SPEC.md`.** `PRODUCT_SPEC.md` is the MVP spec. This document only catalogs capabilities deliberately deferred from MVP. A capability not listed here and not listed in `PRODUCT_SPEC.md` is not in scope.

**Maintenance.** Items are added here when a section review in `PRODUCT_SPEC.md` decides a capability is worth building eventually but is not needed for V1. Each entry records the section it relates to, the idea, why it was deferred, and the date it was deferred.

---

## Deferred Items

### Project templates and starter kits

- **Related section:** §4 Project Lifecycle (with follow-on impacts on §11 Document Generation and Branding and §19 Roles, Permissions, and Collaboration Behavior).
- **Idea.** The Firm Administrator maintains reusable project templates (for example, "Standard Greenfield Sales Cloud Implementation"). A new project created from a template would inherit a predefined set of epics, deliverable templates, or engagement-type-specific defaults.
- **Why deferred.** MVP deliberately drops default starter content on project creation; scope should emerge from discovery rather than from assumed templates. Templates add value once the firm has repeatable engagement patterns and real usage data to template from.
- **Date deferred:** 2026-04-16.

### Consultant transition welcome brief

- **Related section:** §13 Project Knowledge Base.
- **Idea.** When a consultant rolls off a project mid-flight and another joins, the application auto-generates a "welcome brief" from the accumulated project knowledge base so the incoming person can get oriented quickly.
- **Why deferred.** Real consultant transitions can be handled case-by-case in MVP using existing AI synthesis and briefing capabilities. A dedicated generated brief is polish rather than core behavior.
- **Date deferred:** 2026-04-16.
